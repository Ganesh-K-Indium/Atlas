## Intuition

Language is discrete, symbolic, and highly context-dependent — the same word means different things in different sentences, and meaning depends on long-range dependencies a fixed context window has to capture ("it" three sentences later referring back to a noun at the start of a paragraph). Natural Language Processing is the field concerned with building the numerical pipeline that turns raw text into representations a model can compute over, and the architectures that exploit those representations to do something useful: classify, translate, summarize, answer, or generate.

Almost the entire modern history of NLP can be told as progressively better ways of capturing *context* — bag-of-words models that ignored order and context entirely, then recurrent models that processed text sequentially and carried a hidden state forward, then the Transformer, which discards sequential processing altogether and instead lets every token attend directly to every other token in the sequence, making long-range dependencies a single computational hop rather than a chain of many.

<Callout type="tip" title="The one-sentence idea">
  NLP is the problem of representing discrete, ambiguous, context-dependent symbols (words, subwords) as vectors that a model can compute over — and the entire architectural history of the field is a series of better answers to "how much context should each representation account for, and how cheaply?"
</Callout>

## Core Problem Formulations

- **Tokenization:** splitting raw text into discrete units (tokens) a model can embed, since neural networks operate on numeric vectors, not raw character strings.
- **Language modeling:** predicting the probability of the next token given previous context, $p(x_t \mid x_{<t})$ — the pretraining objective underlying essentially all modern generative language models.
- **Sequence classification:** mapping a whole sequence (sentence, document) to a label — sentiment, topic, intent.
- **Token-level tasks:** assigning a label to every token — named entity recognition, part-of-speech tagging.
- **Sequence-to-sequence tasks:** mapping one variable-length sequence to another — machine translation, summarization, question answering with free-form answers.

## Key Techniques and How They Connect

**Tokenization** is the first and most consequential preprocessing step: text must be split into a finite vocabulary of subword units before it can be embedded. Byte-Pair Encoding (BPE) and WordPiece iteratively merge the most frequent adjacent symbol pairs into new vocabulary entries, striking a balance between word-level tokens (too large a vocabulary, poor handling of rare/unseen words) and character-level tokens (too long a sequence, weak per-token semantic content). Subword tokenization lets a model represent any string as a sequence of known pieces, even words it never saw during training, by falling back to smaller fragments. See [Tokenizers](/docs/llms/tokenization-representations/tokenizers) (roadmap).

**The Transformer architecture** replaced recurrence with self-attention: every token computes a weighted combination of every other token's representation, with weights determined by learned query-key-value projections. Because attention is computed in parallel across all positions rather than sequentially, Transformers train far faster than RNNs on the same hardware, and their per-layer path length between any two tokens is constant rather than growing with sequence distance — the key reason they capture long-range dependencies more reliably. See [The Transformer Architecture](/docs/deep-learning/attention-transformers/transformer-architecture) for the full attention derivation.

**Encoder-only models (BERT family)** are trained with a bidirectional objective — masked language modeling, where random tokens are hidden and the model must predict them using context from *both* directions simultaneously. This bidirectionality makes encoder models well suited to understanding tasks (classification, extractive QA, NER) where the full input is available at once, but unsuitable for free-form generation, since generation requires producing tokens one at a time without seeing the future. See [BERT and Bidirectional Encoders](/docs/deep-learning/attention-transformers/bert) (roadmap).

**Decoder-only models (GPT family)** use causal (unidirectional) self-attention: each token can only attend to itself and earlier tokens, enforced by masking out future positions in the attention computation. This matches the autoregressive generation process exactly — at inference time, tokens genuinely are produced one at a time — which is why decoder-only architectures dominate open-ended text generation and modern large language models. See [GPT and Decoder-Only Models](/docs/deep-learning/attention-transformers/gpt-decoder-only) (roadmap).

**Encoder-decoder models** (the original Transformer, T5-style architectures) keep both halves: a bidirectional encoder reads the full source sequence, and a causal decoder generates the target sequence while cross-attending to the encoder's output. This structure maps naturally onto tasks with a clear source/target asymmetry, like translation (source sentence to target-language sentence) and abstractive summarization (long document to short summary).

## Practical Example

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch

tokenizer = AutoTokenizer.from_pretrained("distilbert-base-uncased-finetuned-sst-2-english")
model = AutoModelForSequenceClassification.from_pretrained(
    "distilbert-base-uncased-finetuned-sst-2-english"
)

inputs = tokenizer("This documentation is unexpectedly delightful.", return_tensors="pt")
with torch.no_grad():
    logits = model(**inputs).logits

probs = torch.softmax(logits, dim=-1)
print(model.config.id2label[probs.argmax().item()], probs.max().item())
```

## Common Mistakes

<ProsCons
  pros={[
    "Subword tokenization elegantly handles rare and out-of-vocabulary words by falling back to smaller known fragments.",
    "Pretraining on large unlabeled text corpora and fine-tuning on small labeled datasets (transfer learning) works remarkably well and has replaced training NLP models from scratch.",
    "A single decoder-only LLM can now perform many tasks that once required separate specialized models, via prompting alone.",
