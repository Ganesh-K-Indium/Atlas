## Intuition

A digital image is nothing but a grid of numbers — pixel intensities across height, width, and color channels. There is no explicit concept of "edge," "cat," or "person" anywhere in that array. Computer vision is the applied field concerned with building models that map from this raw numerical grid to the kind of structured understanding a human gets instantly and effortlessly by looking: what objects are present, where they are, what they're doing, and how the scene is organized.

What makes vision hard is not the mapping itself — deep networks are good function approximators — but the *invariances* the mapping must respect. The same cat is still a cat shifted three pixels to the left, rotated ten degrees, partially occluded by a couch, or lit differently. Decades of vision research, arguably summarized by the shift from hand-engineered features (edges, corners, SIFT descriptors) to learned convolutional filters to learned attention over image patches, is really the story of finding architectures whose inductive biases match these invariances well enough to generalize from finite labeled data.

<Callout type="tip" title="The one-sentence idea">
  Computer vision is the search for representations of pixel grids that are stable under the transformations that don't change meaning (translation, scale, lighting, minor occlusion) while remaining sensitive to the transformations that do (a different object, a different pose, a different scene).
</Callout>

## Core Problem Formulations

Vision is not one task — it's a family of related tasks operating at different levels of spatial granularity, and most of what looks like "progress in vision" is really progress at pushing predictions to finer and finer spatial resolution:

- **Image classification:** map an entire image to a single label (or label distribution) — "what is the dominant subject of this image?"
- **Object detection:** predict a set of bounding boxes, each with a class label and confidence score — "what objects are present, and roughly where?"
- **Semantic segmentation:** classify *every pixel* into a category, with no distinction between separate instances of the same class.
- **Instance segmentation:** like semantic segmentation, but separates distinct object instances (two overlapping cars get two different masks, not one merged blob).
- **Image generation / synthesis:** the inverse problem — mapping from a latent code, noise, or text description to a plausible image.
- **Vision-language grounding:** joint tasks connecting pixels to natural language — image captioning, visual question answering, text-to-region retrieval.

Formally, classification is a function $f_\theta: \mathbb{R}^{H \times W \times C} \to \Delta^{K-1}$ (a probability simplex over $K$ classes), while detection and segmentation are *structured prediction* problems — the output is a variable-size set of boxes or a dense per-pixel labeling, not a fixed-size vector, which is precisely why they need architectural machinery beyond a plain classifier head.

## Key Techniques and How They Connect

**Convolutional neural networks** are the foundational architecture of modern vision. A convolutional layer slides a small learned filter across the image, so the same weights are reused at every spatial location — this bakes in translation equivariance for free, which is why CNNs need far less data than a fully-connected network to learn "an edge is an edge wherever it appears." Stacking convolutions with pooling builds a hierarchy: early layers detect edges and textures, middle layers detect parts, deep layers detect whole objects. See [Convolutional Neural Networks](/docs/deep-learning/vision-architectures/cnn) for the full derivation of convolution, receptive fields, and classic architectures (LeNet through ResNet).

**Vision Transformers (ViT)** split an image into fixed-size patches, linearly project each patch into a token embedding, and feed the resulting sequence through a standard Transformer encoder — treating an image exactly like a sentence of "patch tokens." This trades the convolution's built-in translation-equivariance prior for a much weaker one (attention can look anywhere from the start), which is why ViTs typically need larger pretraining datasets than CNNs to reach competitive accuracy, but often surpass CNNs once that data is available. See [Vision Transformers](/docs/deep-learning/vision-architectures/vision-transformers) (roadmap).

**Object detection** architectures add machinery on top of a CNN or ViT backbone to produce a variable number of boxes. Two-stage detectors (e.g. Faster R-CNN) first propose candidate regions, then classify and refine each one. Single-stage detectors like the YOLO family instead divide the image into a grid and directly regress box coordinates and class probabilities per grid cell in one forward pass, trading a little accuracy for large speed gains — the reason YOLO variants dominate real-time detection. See [YOLO and Single-Stage Detectors](/docs/deep-learning/vision-architectures/yolo) (roadmap).

**Segmentation** has recently been reshaped by promptable, foundation-model approaches: rather than training a fixed-class segmentation network per dataset, a model like Segment Anything is trained to take a point, box, or text prompt and produce a high-quality mask for the object being pointed at, generalizing zero-shot to object categories never seen during training. See [Segment Anything and Promptable Segmentation](/docs/deep-learning/vision-architectures/segment-anything) (roadmap).

**Vision-language models** like CLIP jointly embed images and text captions into a shared vector space using a contrastive objective, so that an image and its matching caption end up close together while mismatched pairs are pushed apart. This shared embedding space underlies zero-shot image classification (score an image against a set of candidate label sentences), image-text retrieval, and is the visual backbone used inside many multimodal LLMs. See [CLIP and Contrastive Vision-Language Pretraining](/docs/deep-learning/vision-architectures/clip) (roadmap).

  headers={["Approach", "Best for", "Data / compute needs", "Key weakness"]}

## Common Mistakes

<ProsCons
  pros={[
    "Transfer learning from ImageNet- or web-scale-pretrained backbones works extremely well even on small downstream datasets, dramatically lowering the labeled-data bar.",
    "Data augmentation (crops, flips, color jitter, mixup) is cheap and consistently improves generalization because it directly encodes the invariances the task cares about.",
    "Modern foundation vision models generalize zero-shot to categories never explicitly trained on.",
  ]}
  cons={[
    "Evaluating only on overall accuracy hides severe class imbalance problems — a detector can score well in aggregate while missing every instance of a rare but critical class (e.g., a pedestrian at night).",
    "Training/test distribution shift (different camera, lighting, geography) silently tanks real-world performance even when validation-set metrics look excellent.",
    "Treating segmentation/detection purely as classification-with-more-steps ignores that these are structured prediction problems — box regression, non-max suppression, and mask coherence all introduce failure modes classification doesn't have.",
    "Resizing/cropping preprocessing choices are easy to get inconsistent between training and inference, silently corrupting spatial predictions like bounding boxes.",
  ]}
