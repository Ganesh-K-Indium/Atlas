## Intuition

A conventional segmentation model like [U-Net](/docs/deep-learning/vision-architectures/unet) is trained for a fixed set of classes: it always outputs, say, "tumor vs. background," or "road / car / pedestrian / sky," and asking it to segment some object type it never saw during training simply doesn't work — the model's entire output structure is tied to the label set it was trained on. SAM's reframing: instead of training a model that always segments the *same fixed things*, train a model that segments *whatever it's currently told to look at*, where "told" can be a point, a box, or a rough mask — a **prompt** given at inference time, exactly analogous to how a language model is prompted with a task description rather than retrained for it.

<Callout type="tip" title="The one-sentence idea">
  Split segmentation into a heavy, prompt-independent image encoder (run once per image) and a lightweight, fast prompt encoder + mask decoder that combines the image's precomputed embedding with whatever prompt (point, box, rough mask) is given at that moment — decoupling "understand this image in general" from "produce the mask for this specific thing right now" is what lets the model generalize to object types and domains it never saw labeled examples for.
</Callout>

## Promptable Segmentation Architecture

SAM has three components with deliberately different computational costs:

1. **Image encoder** — a large, pretrained ViT (see [Vision Transformers (ViT)](/docs/deep-learning/vision-architectures/vision-transformers)) that processes the full input image *once* into a rich, general-purpose dense embedding. This is the expensive step (hundreds of milliseconds), but it is prompt-independent — it doesn't know or care yet what will be segmented, so it only ever needs to run once per image, no matter how many different prompts/objects are later queried against that same image.
2. **Prompt encoder** — encodes whatever prompt is given into an embedding: a point click is encoded via its position (positional encoding, as in a Transformer) plus a learned embedding indicating foreground/background; a box is encoded via its corner positions; a rough mask is encoded via a small convolutional embedding. This is cheap, since prompts are tiny compared to the image itself.
3. **Mask decoder** — a lightweight Transformer-based decoder that takes the (precomputed, reused) image embedding and the (freshly computed, cheap) prompt embedding, cross-attends between them, and outputs a segmentation mask (plus a confidence/IoU-prediction score, and — because a single point can be ambiguous, e.g. a click on a shirt button could mean "the button," "the shirt," or "the person" — SAM by default outputs multiple candidate masks at different granularities and lets the confidence score pick, or a user disambiguate).

Because only the prompt encoder and mask decoder need to re-run for a new prompt on an *already-encoded* image, SAM can respond to a new point or box interactively, in real time (tens of milliseconds), even though the image encoder itself is a large, slow-to-run ViT — the expensive part is amortized once per image, and every subsequent prompt is cheap.

## Why Promptable Design Generalizes to Unseen Objects and Domains

A conventional segmentation model's output layer is shaped by, and only by, the fixed class taxonomy present in its training labels — it has no mechanism at all for producing a coherent mask for an object category it never saw, because "which class is this pixel" was the entire task it learned. SAM instead never learns "what class is this" at all — it learns a much more general skill: **given a spatial hint about what region a user cares about, produce the boundary of the coherent object/region containing that hint.** That skill is class-agnostic by construction; it is trained on masks, not class labels, using a large and diverse (reportedly over 1 billion masks across 11 million images) dataset built specifically to cover a very broad range of object types, scales, and domains — precisely so the "given a hint, find the object boundary" skill transfers to object categories and imaging domains not explicitly represented in training.

This mirrors, structurally, why [CLIP](/docs/deep-learning/vision-architectures/clip) generalizes zero-shot to unseen classes: both replace "predict one of K fixed labels" with a more general, prompt/query-driven task (compare embeddings to arbitrary text, versus produce a mask for an arbitrary spatial hint) whose training signal is not tied to a closed set of categories. The result in both cases is a foundation model that can be *directed* at inference time rather than *retrained* for each new task.

## Code Sketch

```python
import torch

class SAMLike(torch.nn.Module):
    def __init__(self, image_encoder, prompt_encoder, mask_decoder):
        super().__init__()
        self.image_encoder = image_encoder   # large ViT, expensive, run once per image
        self.prompt_encoder = prompt_encoder  # cheap: encodes points/boxes/masks
        self.mask_decoder = mask_decoder      # lightweight, cross-attends image + prompt

    def encode_image(self, image):
        return self.image_encoder(image)      # (1, embed_dim, H', W') — cache and reuse

    def predict_mask(self, image_embedding, points=None, boxes=None, mask_prompt=None):
        prompt_embedding = self.prompt_encoder(points=points, boxes=boxes, mask=mask_prompt)
        masks, iou_scores = self.mask_decoder(image_embedding, prompt_embedding)
        return masks, iou_scores    # multiple candidate masks + confidence, since prompts can be ambiguous

# Usage: encode once, prompt many times cheaply
sam = SAMLike(image_encoder, prompt_encoder, mask_decoder)
image_embedding = sam.encode_image(image)                       # expensive, once
masks_a, scores_a = sam.predict_mask(image_embedding, points=[(120, 80)])   # cheap
masks_b, scores_b = sam.predict_mask(image_embedding, boxes=[(50, 50, 200, 300)])  # cheap, same image
```
