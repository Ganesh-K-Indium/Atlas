## Intuition

Once CNNs proved they worked, the obvious next question was: does stacking more layers keep helping? Empirically, no — beyond a certain depth, plain stacks of convolutions get *worse* on both training and test error, not just test error. That rules out overfitting as the explanation; something about optimizing very deep plain networks is fundamentally harder. This is the **degradation problem**, and it's the problem ResNet was built to solve. DenseNet and EfficientNet then push the same underlying question — how do you get more representational power out of a CNN without the optimization or efficiency cost — in different directions: DenseNet maximizes feature reuse, EfficientNet asks how to scale a good architecture up rationally.

<Callout type="tip" title="The one-sentence idea">
  If a deeper network can always fall back to computing the identity function on its extra layers, then adding depth should never make training error worse in principle — the degradation problem shows that plain stacked layers can't easily learn to *be* the identity even when that's the optimal thing to do, and ResNet's shortcut connection removes that difficulty by construction.
</Callout>

## ResNet: Residual Connections Solve the Degradation Problem

Consider a plain block of stacked layers that must learn some target mapping $H(x)$ from input $x$. A residual block instead parameterizes the *same* layers to learn the **residual** $F(x) = H(x) - x$, and reconstructs the desired output by adding the input back:

$$
y = F(x, \{W_i\}) + x
$$

Why this matters, precisely:

1. **Trivial identity fallback.** If the optimal transformation for a block truly is the identity (i.e., extra depth adds nothing useful at this point in the network), a residual block only needs to drive $F(x) \to 0$ — pushing a stack of weights to zero is a much easier optimization target for gradient descent than forcing an arbitrary stack of nonlinear layers to reproduce $x$ exactly. A plain (non-residual) block would need its weights to conspire *exactly* into an identity mapping through nonlinearities, which is a much harder point to find and to stay near.
2. **Direct gradient path.** Differentiating $y = F(x) + x$ with respect to $x$ gives $\frac{\partial y}{\partial x} = \frac{\partial F}{\partial x} + I$. The `+ I` term means that no matter how small or poorly conditioned $\frac{\partial F}{\partial x}$ becomes, gradient signal from later layers is guaranteed a direct, unimpeded path (via the shortcut) back to earlier layers, un-multiplied by any of the block's weights. Chained across $N$ residual blocks, the gradient contains a sum of terms including the "all shortcuts, no residual path" term, which never vanishes or explodes regardless of depth. This is the same structural fix that residual connections provide inside Transformer blocks (see [The Transformer Architecture](/docs/deep-learning/attention-transformers/transformer-architecture)).
3. **Empirical resolution of degradation.** Because of (1) and (2), residual networks with 50, 101, even 152+ layers train to *lower* training error than their shallower counterparts — the degradation problem disappears once the identity shortcut is available as a free, always-accessible path.

When the input and output channel dimensions differ (e.g. at a stage transition), the shortcut uses a $1\times1$ convolution to project $x$ to matching dimensions before the addition, rather than a raw identity.

## DenseNet: Dense Connectivity

ResNet connects each block's output to the *next* block via addition. DenseNet goes further: within a "dense block," **every layer receives the concatenation of the feature maps of all preceding layers** as its input, and its own output is concatenated onto that growing stack for every subsequent layer:

$$
x_\ell = H_\ell\big([x_0,\ x_1,\ \ldots,\ x_{\ell-1}]\big)
$$

where $[x_0, \ldots, x_{\ell-1}]$ denotes channel-wise concatenation (not addition) of all previous layers' feature maps, and $H_\ell$ is a small composite function (BatchNorm → ReLU → Conv). Each layer $H_\ell$ only needs to produce a small number of new feature maps (the "growth rate" $k$, often as small as 12–32), because it can freely reuse every feature already computed by earlier layers rather than needing to recompute or re-derive them.

This has several consequences:

- **Maximal feature reuse and gradient flow**: every layer has a direct connection to every earlier layer's output (and, via backprop, to the loss from every later layer), which — like ResNet's shortcut, but even more directly — alleviates vanishing gradients.
- **Parameter efficiency**: because layers stay narrow (small growth rate) and reuse existing features instead of re-learning them, DenseNets can match ResNet-level accuracy with substantially fewer parameters and FLOPs.
- **Growing input width**: input channels to $H_\ell$ grow linearly with depth ($\ell \times k$ roughly), which is why DenseNets periodically insert "transition layers" ($1\times1$ conv + pooling) between dense blocks to compress channels and downsample spatially — otherwise the concatenation would grow unboundedly.

## EfficientNet: Compound Scaling

Given a base CNN architecture, there are three independent ways to make it "bigger": add more layers (**depth** $d$), make each layer wider — more channels (**width** $w$), or process higher-resolution images (**resolution** $r$). Prior work typically scaled just one of these arbitrarily (e.g. ResNet-50 → ResNet-152 scales depth only). EfficientNet's key empirical finding is that scaling only one dimension hits diminishing returns quickly, and that scaling all three **jointly**, in a fixed ratio, is far more accuracy-efficient.

Compound scaling defines a single coefficient $\phi$ (that a practitioner chooses to trade off compute budget vs. accuracy) and scales all three dimensions by constants raised to that power:

$$
\text{depth: } d = \alpha^\phi, \qquad \text{width: } w = \beta^\phi, \qquad \text{resolution: } r = \gamma^\phi
$$

subject to $\alpha \cdot \beta^2 \cdot \gamma^2 \approx 2$ and $\alpha, \beta, \gamma \geq 1$, where $\alpha, \beta, \gamma$ are constants found once via a small grid search on the base model. The constraint on $\alpha\beta^2\gamma^2$ comes from the fact that FLOPs scale roughly linearly with depth but quadratically with width and resolution (since both spatial dimensions of a feature map scale with resolution, and a convolution's cost scales as the product of input and output channels for width) — so this constraint keeps total compute increasing by approximately $2^\phi$ for a given $\phi$, giving a predictable compute budget as $\phi$ increases.

The intuition for *why* jointly scaling helps: a higher-resolution input has more pixels, so it needs a deeper network to grow the receptive field enough to cover a proportionally larger image, and it needs more channels per layer to capture the finer-grained detail the extra resolution provides. Scaling only depth on a high-resolution image leaves the receptive-field-per-pixel-detail balance off; scaling all three keeps that balance consistent as the model grows, which is why the compound-scaled EfficientNet family achieves better accuracy-per-FLOP than models produced by scaling any single dimension.

## Code Sketch

```python
import torch
import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn1 = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1)
        self.bn2 = nn.BatchNorm2d(channels)
        self.relu = nn.ReLU(inplace=True)

    def forward(self, x):
        identity = x
        out = self.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))
        out = out + identity          # the shortcut: F(x) + x
        return self.relu(out)

class DenseLayer(nn.Module):
    def __init__(self, in_channels, growth_rate):
        super().__init__()
        self.bn = nn.BatchNorm2d(in_channels)
        self.conv = nn.Conv2d(in_channels, growth_rate, 3, padding=1)

    def forward(self, x):
        out = self.conv(torch.relu(self.bn(x)))
        return torch.cat([x, out], dim=1)   # concatenate, don't add

class DenseBlock(nn.Module):
    def __init__(self, in_channels, growth_rate, num_layers):
        super().__init__()
        layers = []
        channels = in_channels
        for _ in range(num_layers):
            layers.append(DenseLayer(channels, growth_rate))
            channels += growth_rate   # input width grows each layer
        self.layers = nn.Sequential(*layers)

    def forward(self, x):
        return self.layers(x)
```
