## Intuition

Object detection needs two things at once: *where* are the objects (bounding boxes), and *what* are they (class labels) — for a variable, unknown number of objects per image. The natural-seeming approach is to do this in two stages: first propose candidate regions that might contain an object, then run a classifier on each proposed region. That works, but running a full classification pass over hundreds or thousands of candidate regions per image is slow — nowhere near fast enough for real-time video.

YOLO's ("You Only Look Once") bet: skip the separate proposal stage entirely. Look at the whole image *once*, and have a single CNN forward pass directly output every bounding box and class prediction simultaneously, framed as one big regression problem.

<Callout type="tip" title="The one-sentence idea">
  Divide the image into an S x S grid, and have one forward pass through a single CNN predict, for every grid cell, a fixed number of candidate bounding boxes plus class probabilities — turning detection into one regression problem over a fixed-size output tensor, instead of a separate propose-then-classify pipeline run over a variable number of region proposals.
</Callout>

## Single-Shot Detection Mechanism

YOLO divides the input image into an $S \times S$ grid. Each grid cell is made responsible for detecting any object whose center falls inside it. For every cell, the network predicts, in a single forward pass:

- $B$ candidate bounding boxes, each parameterized as $(x, y, w, h, \text{confidence})$ — $(x, y)$ the box center relative to the cell, $(w, h)$ width/height relative to the whole image, and confidence an estimate of $\Pr(\text{object}) \times \text{IoU}_{\text{pred, truth}}$ (how likely a box is to contain an object, scaled by how well it would overlap the ground truth if so).
- $C$ conditional class probabilities $\Pr(\text{class}_i \mid \text{object})$, shared across the boxes predicted by that cell.

The entire output is one tensor of shape $S \times S \times (B \cdot 5 + C)$, produced by one CNN forward pass over the whole image — there is no separate stage that first crops out candidate regions and re-runs a network on each one. The network is trained end-to-end with a multi-part regression loss (localization error on box coordinates, confidence error, classification error), all computed directly against this fixed-size grid tensor.

## Two-Stage vs. One-Stage Detectors

Two-stage detectors (e.g. the R-CNN family) split the problem explicitly: stage one, a **region proposal** mechanism (originally external algorithms like selective search, later a learned Region Proposal Network in Faster R-CNN) generates a set of candidate boxes likely to contain an object, without yet knowing what class; stage two, each proposed region is cropped, resized, and passed through a classifier (plus a box-refinement regressor) to assign a class label and adjust the box. This is more accurate (proposals let the classifier focus compute on promising regions, and box refinement happens twice), but requires running the classification stage once per proposal — hundreds of forward passes over cropped regions per image — which is why two-stage detectors historically ran far slower than YOLO's single forward pass over the whole image.

## Non-Max Suppression

Because every one of the $S \times S$ cells (and every one of its $B$ boxes) makes an independent prediction, a single real object very often gets multiple overlapping candidate boxes — nearby cells frequently all "notice" the same object and each propose a box for it. Left alone, this would report the same object multiple times. **Non-max suppression (NMS)** cleans this up:

1. Discard all boxes below a confidence threshold.
2. Take the remaining box with the highest confidence score; keep it as a final detection.
3. Compute Intersection-over-Union (IoU) between that box and every other remaining box *of the same predicted class*; suppress (discard) any box whose IoU with it exceeds a threshold (e.g. 0.5) — these are judged to be duplicate detections of the same object.
4. Repeat from step 2 with the remaining boxes, until none are left.

The IoU threshold is what distinguishes "this is a second, genuinely separate nearby object" (low IoU with the kept box — retained) from "this is a duplicate detection of the object we already kept" (high IoU — suppressed). Without NMS, a single-shot detector's raw output would contain many redundant, overlapping boxes per real object, since nothing in the grid-based prediction mechanism inherently prevents that.

## Code Sketch

```python
import torch

def non_max_suppression(boxes, scores, classes, iou_threshold=0.5):
    """boxes: (N, 4) as (x1, y1, x2, y2); scores, classes: (N,)"""
    keep = []
    order = scores.argsort(descending=True)
    while order.numel() > 0:
        i = order[0].item()
        keep.append(i)
        if order.numel() == 1:
            break
        rest = order[1:]
        # only compare boxes of the same predicted class
        same_class = classes[rest] == classes[i]
        ious = compute_iou(boxes[i], boxes[rest])           # (len(rest),)
        suppress = (ious > iou_threshold) & same_class
        order = rest[~suppress]
    return keep

class YOLOHead(torch.nn.Module):
    """Predicts the S x S x (B*5 + C) output tensor from backbone features."""
    def __init__(self, in_channels, grid_size=7, num_boxes=2, num_classes=20):
        super().__init__()
        self.S, self.B, self.C = grid_size, num_boxes, num_classes
        out_channels = grid_size * grid_size * (num_boxes * 5 + num_classes)
        self.fc = torch.nn.Linear(in_channels, out_channels)

    def forward(self, features):
        out = self.fc(features)
        return out.view(-1, self.S, self.S, self.B * 5 + self.C)
```
