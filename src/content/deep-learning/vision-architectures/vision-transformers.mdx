## Intuition

A Transformer encoder operates on a *sequence of tokens* — that's the entire interface (see [The Transformer Architecture](/docs/deep-learning/attention-transformers/transformer-architecture)). Applying self-attention to every individual pixel of even a modest $224\times224$ image would mean a sequence of 50,176 tokens, and self-attention's $O(n^2)$ cost makes that completely impractical. ViT's move is almost embarrassingly simple: don't tokenize pixels — tokenize small square **patches** of the image instead, treating each patch as one "word" in a visual sentence.

<Callout type="tip" title="The one-sentence idea">
  Split the image into a grid of fixed-size patches, flatten and linearly project each patch into a token embedding, prepend a position embedding and a learnable [CLS] token, and feed the whole sequence into a standard Transformer encoder — an image becomes a sequence, and everything downstream is identical to a text Transformer.
</Callout>

## Patch Embedding — Turning an Image into a Sequence

Given an image $x \in \mathbb{R}^{H \times W \times C}$, ViT partitions it into a grid of $N$ non-overlapping patches of size $P \times P$, where $N = \frac{H \times W}{P^2}$. Each patch is flattened into a vector of length $P^2 \cdot C$, then linearly projected (a single learned matrix $E$, shared across all patches — this is a convolution with kernel size $P$ and stride $P$, in effect) into a $D$-dimensional embedding:

$$
z_0 = [\,x_{\text{class}};\ x_p^1 E;\ x_p^2 E;\ \ldots;\ x_p^N E\,] + E_{\text{pos}}
$$

where $x_p^i$ is the flattened $i$-th patch, $x_{\text{class}}$ is a learnable token prepended to the sequence (analogous to BERT's `[CLS]`, whose final-layer representation is used for classification), and $E_{\text{pos}}$ is a learned positional embedding added to every token — necessary because, exactly as in a text Transformer, self-attention itself has no notion of position, and patch order/location must be injected explicitly.

The resulting sequence of $N+1$ vectors is fed directly into a standard pre-norm Transformer encoder — the same multi-head self-attention and feed-forward blocks used for text, entirely unmodified. The final [CLS] token representation is passed through a small MLP head for classification. This is the whole architecture: no convolutions anywhere except the (mathematically equivalent-to-linear) patch projection.

## Why ViT Needs More Data or Regularization Than CNNs

A CNN's convolutional structure builds in two **inductive biases** for free, purely from the architecture, before any training happens:

- **Locality**: a filter only ever looks at a small local patch, encoding the assumption that nearby pixels are related.
- **Translation equivariance**: the same filter is applied everywhere (parameter sharing, see [Convolutional Neural Networks](/docs/deep-learning/vision-architectures/cnn)), encoding the assumption that a pattern means the same thing regardless of where it appears.

A ViT has neither bias architecturally. Self-attention lets every patch attend to every other patch from the very first layer, uniformly — there's no built-in notion that nearby patches are more likely to be related, and the shared linear patch projection provides some translation equivariance for the token embeddings themselves, but the attention mechanism that mixes tokens together has to *learn*, from data, whatever locality or spatial structure happens to be useful. This makes ViT strictly more flexible (it isn't restricted to only ever learning local, translation-equivariant functions) but also means it must learn from data something a CNN gets for free from its architecture.

The empirical consequence reported in the original paper: trained on a mid-sized dataset like ImageNet-1k alone, ViT underperforms comparable ResNets, because it hasn't seen enough examples to learn locality and spatial structure on its own and has no architectural shortcut supplying it. Trained on much larger datasets (e.g. JFT-300M, hundreds of millions of images) or with strong data augmentation/regularization, ViT matches or exceeds CNN performance — with enough data, the flexibility to learn *whatever* attention pattern the task actually needs (not necessarily local, not necessarily translation-only) becomes an advantage rather than a liability. This is the same "biases help with less data, hurt the ceiling with enough data" trade-off that shows up whenever an inductive bias is compared against a more general, less-constrained model class.

## Code Sketch

```python
import torch
import torch.nn as nn

class PatchEmbedding(nn.Module):
    def __init__(self, img_size=224, patch_size=16, in_channels=3, embed_dim=768):
        super().__init__()
        self.num_patches = (img_size // patch_size) ** 2
        # A conv with kernel = stride = patch_size implements "flatten patch, project"
        self.proj = nn.Conv2d(in_channels, embed_dim, kernel_size=patch_size, stride=patch_size)

    def forward(self, x):
        x = self.proj(x)                     # (batch, embed_dim, H/P, W/P)
        x = x.flatten(2).transpose(1, 2)      # (batch, num_patches, embed_dim)
        return x

class ViT(nn.Module):
    def __init__(self, img_size=224, patch_size=16, embed_dim=768, depth=12, num_heads=12, num_classes=1000):
        super().__init__()
        self.patch_embed = PatchEmbedding(img_size, patch_size, 3, embed_dim)
        num_patches = self.patch_embed.num_patches

        self.cls_token = nn.Parameter(torch.zeros(1, 1, embed_dim))
        self.pos_embed = nn.Parameter(torch.zeros(1, num_patches + 1, embed_dim))

        encoder_layer = nn.TransformerEncoderLayer(embed_dim, num_heads, dim_feedforward=embed_dim * 4)
        self.encoder = nn.TransformerEncoder(encoder_layer, num_layers=depth)
        self.head = nn.Linear(embed_dim, num_classes)

    def forward(self, x):
        x = self.patch_embed(x)                          # (batch, N, D)
        cls_tokens = self.cls_token.expand(x.shape[0], -1, -1)
        x = torch.cat([cls_tokens, x], dim=1)             # prepend [CLS]
        x = x + self.pos_embed                            # inject position information
        x = self.encoder(x)
        return self.head(x[:, 0])                         # classify from [CLS] token
```
