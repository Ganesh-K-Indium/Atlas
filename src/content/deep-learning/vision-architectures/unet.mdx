## Intuition

Image classification only needs to answer one question per image ("what is this?"), so a CNN can safely throw away spatial precision as it goes deeper — by the final layers, all that matters is *whether* a cat-like pattern was detected anywhere, not exactly which pixels it occupied. Segmentation needs the opposite: a label for *every single pixel*. That's in direct tension with how CNNs normally work — pooling and strided convolutions downsample specifically to build large receptive fields and abstract, position-blurred features, discarding exactly the fine-grained spatial detail a pixel-level task needs.

U-Net's answer: run a normal encoder that downsamples and abstracts (to know *what* is where, at a coarse level), then a decoder that upsamples back to full resolution (to produce a full-resolution output map) — but wire in direct connections from each encoder resolution to the matching decoder resolution, so the fine spatial detail lost during downsampling can flow straight across instead of having to be reconstructed from scratch by the decoder.

<Callout type="tip" title="The one-sentence idea">
  The encoder answers "what is present, roughly where" by discarding spatial detail as it deepens; the decoder needs that detail back to draw precise boundaries, and skip connections deliver it directly from the encoder layer of matching resolution rather than forcing the decoder to hallucinate it from a heavily downsampled bottleneck.
</Callout>

## Encoder-Decoder with Skip Connections

The **encoder** (contracting path) is a standard CNN stack: repeated (conv → ReLU) blocks followed by downsampling (max-pool or strided conv), roughly doubling the channel count and halving spatial resolution at each stage. As in any CNN, this builds a hierarchy from local, precise, low-level features (early layers) to global, abstract, spatially coarse features (the bottleneck).

The **decoder** (expanding path) mirrors this in reverse: repeated upsampling (transposed convolution or interpolation + conv) followed by convolutional blocks, roughly halving channel count and doubling spatial resolution at each stage, until the output matches the input's spatial resolution.

If the decoder only had the bottleneck to work with, it would need to reconstruct exact object boundaries from a representation that has already discarded most spatial precision in favor of "what is present" abstraction — an under-determined, lossy inverse problem. U-Net's **skip connections** solve this directly: at each decoder stage, the upsampled feature map is concatenated (channel-wise) with the encoder feature map from the *same spatial resolution*, before the next convolution:

$$
D_\ell = \text{Conv}\big([\text{Upsample}(D_{\ell+1}),\ E_\ell]\big)
$$

where $E_\ell$ is the encoder's feature map at resolution level $\ell$ and $D_{\ell+1}$ is the decoder's (coarser) output from one level deeper. Because $E_\ell$ was captured *before* that resolution level's information was discarded by further downsampling, it still contains the precise edge and texture information needed to place a segmentation boundary accurately — the network doesn't need to reconstruct that detail, only to combine it with the coarse "what is this region" signal coming up from the bottleneck.

This is the same underlying idea as a ResNet shortcut (give a later layer direct access to an earlier representation instead of forcing it through an information bottleneck) but concatenated rather than added, and connecting matched-resolution stages across an encoder-decoder rather than connecting a block to its own output.

## Loss and Output

The final layer is a $1\times1$ convolution mapping the decoder's final feature map to $C$ channels (one per class), followed by a per-pixel softmax; the network is trained with per-pixel cross-entropy (often combined with a Dice loss to directly optimize the overlap between predicted and ground-truth masks, since cross-entropy alone treats every pixel independently and can be dominated by large background classes in imbalanced masks). Because every operation is applied convolutionally (no fully-connected layers), the same trained U-Net can be applied to input images of any size at inference time.

## Code Sketch

```python
import torch
import torch.nn as nn

class ConvBlock(nn.Module):
    def __init__(self, in_ch, out_ch):
        super().__init__()
        self.block = nn.Sequential(
            nn.Conv2d(in_ch, out_ch, 3, padding=1), nn.ReLU(inplace=True),
            nn.Conv2d(out_ch, out_ch, 3, padding=1), nn.ReLU(inplace=True),
        )

    def forward(self, x):
        return self.block(x)

class UNet(nn.Module):
    def __init__(self, in_ch=3, num_classes=2, base=64):
        super().__init__()
        self.enc1 = ConvBlock(in_ch, base)
        self.enc2 = ConvBlock(base, base * 2)
        self.pool = nn.MaxPool2d(2)
        self.bottleneck = ConvBlock(base * 2, base * 4)

        self.up2 = nn.ConvTranspose2d(base * 4, base * 2, 2, stride=2)
        self.dec2 = ConvBlock(base * 4, base * 2)   # base*4 = upsampled + skip concat
        self.up1 = nn.ConvTranspose2d(base * 2, base, 2, stride=2)
        self.dec1 = ConvBlock(base * 2, base)

        self.out_conv = nn.Conv2d(base, num_classes, kernel_size=1)

    def forward(self, x):
        e1 = self.enc1(x)                 # full resolution features
        e2 = self.enc2(self.pool(e1))     # 1/2 resolution features
        b = self.bottleneck(self.pool(e2))  # 1/4 resolution, most abstract

        d2 = self.up2(b)
        d2 = self.dec2(torch.cat([d2, e2], dim=1))   # skip connection: fine detail from e2
        d1 = self.up1(d2)
        d1 = self.dec1(torch.cat([d1, e1], dim=1))   # skip connection: fine detail from e1

        return self.out_conv(d1)   # per-pixel class logits, same spatial size as input
```
