## Intuition

Underneath every embedding technique in this history is one idea, stated by linguist J.R. Firth in 1957: **"You shall know a word by the company it keeps."** This is the **distributional hypothesis** — words that tend to occur in similar contexts tend to have similar meanings. "Dog" and "cat" both appear near "pet," "vet," "leash," "food" far more often than either appears near "quantum" or "spreadsheet" — and that statistical pattern alone, with no dictionary definitions involved, is enough to learn that "dog" and "cat" are semantically related.

Every method below is a different way of turning co-occurrence statistics into vectors, and the biggest split in the family tree is: does a word get **one fixed vector for all its uses**, or does its vector **change depending on the sentence it's in**?

<Callout type="tip" title="The one-sentence idea">
  Static embeddings (Word2Vec, GloVe) learn one vector per word type by exploiting which words co-occur across a whole corpus; contextual embeddings (BERT/GPT-style Transformers) compute a fresh vector per word token by letting it attend to its actual surrounding words at inference time — the same underlying distributional signal, captured at two different granularities.
</Callout>

## Static Embeddings: One Vector Per Word

### Word2Vec: Predict Context from Word, or Word from Context

Word2Vec (Mikolov et al., 2013) learns embeddings purely as a byproduct of a simple prediction task — the embeddings are the weights of a shallow network trained to solve it, not the direct target. Two variants:

- **Skip-gram**: given the current word, predict each of its surrounding context words (within a fixed window). The objective, for a corpus of words $w_1, \ldots, w_T$ and context window size $c$:

$$
\mathcal{L}_{\text{skip-gram}} = \sum_{t=1}^{T} \sum_{-c \le j \le c,\, j \ne 0} \log p(w_{t+j} \mid w_t)
$$

- **CBOW (Continuous Bag of Words)**: the mirror image — given the surrounding context words, predict the current word.

$$
p(w_t \mid w_t) \; \text{modeled via} \; p(w_O \mid w_I) = \frac{\exp(v_{w_O}^\top v_{w_I})}{\sum_{w=1}^{V} \exp(v_w^\top v_{w_I})}
$$

Computing that softmax's denominator over the full vocabulary $V$ at every training step is expensive, so in practice Word2Vec is trained with **negative sampling**: instead of normalizing over the whole vocabulary, the model is trained to distinguish the true context word from a handful of randomly sampled "negative" words — turning a $V$-way classification into a small number of binary classifications per step, which is what actually makes training on billion-word corpora tractable.

After training, the input-side weight matrix (the mapping from one-hot word to hidden vector) *is* the embedding table — each row is that word's static vector, and words that appeared in similar contexts end up with similar vectors purely because they had to make similar context predictions.

### GloVe: Factorize the Co-occurrence Matrix Directly

GloVe (Pennington, Socher & Manning, 2014) takes a more direct route to the same distributional signal. Instead of a prediction task, it explicitly builds a **word–word co-occurrence matrix** $X$, where $X_{ij}$ counts how often word $j$ appears in the context of word $i$ across the whole corpus, then learns vectors $w_i, w_j$ (and biases $b_i, b_j$) whose dot product approximates the *log* of that co-occurrence count:

$$
J = \sum_{i,j=1}^{V} f(X_{ij}) \left( w_i^\top \tilde{w}_j + b_i + \tilde{b}_j - \log X_{ij} \right)^2
$$

where $f(X_{ij})$ is a weighting function that down-weights extremely frequent pairs (like "the" with everything) so they don't dominate the loss. This is, in effect, a weighted low-rank matrix factorization of the log co-occurrence matrix — GloVe's embeddings are literally the factors of a global count statistic, whereas Word2Vec's are the incidental weights of a local prediction task. Empirically the two produce embeddings with broadly similar geometric properties (including the famous analogy arithmetic, e.g. king − man + woman ≈ queen).

## The Ceiling of Static Embeddings

A static embedding gives "bank" exactly one vector — the same vector whether it appears in "river bank" or "savings bank." Any polysemy (multiple meanings) or context-dependent nuance simply cannot be represented, because the vector is looked up from a fixed table indexed by word type, with no way to see the sentence around it.

## Contextual Embeddings: A Vector Per Token, Computed From Its Context

Transformer-based language models replace the fixed lookup table with a computation: each token starts from a static embedding (its row in an embedding table, just like before) but that vector is then repeatedly updated by self-attention layers that mix in information from every other token in the sequence. The final hidden state for "bank" in "I deposited money at the bank" and the final hidden state for "bank" in "I sat on the river bank" start from the same row of the embedding table but end up as different vectors, because they attended to different surrounding tokens.

$$
h_i^{(l)} = \text{TransformerLayer}\left(h_i^{(l-1)}, \{h_1^{(l-1)}, \ldots, h_n^{(l-1)}\}\right)
$$

This is still the distributional hypothesis at work — "you shall know a word by the company it keeps" — just applied at inference time to the *specific* company a token keeps in *this* sentence, rather than baked in once as an average over the entire training corpus.

<Callout type="note">
  Static embeddings didn't disappear inside modern LLMs — they're still the very first layer (the token embedding table that turns a token ID into an initial vector). What changed is that this is now only the *starting point*; the representation actually used for prediction is the output of many Transformer layers of context-mixing on top of it.
</Callout>

## Code Sketch

```python
# Static: Word2Vec via gensim
from gensim.models import Word2Vec

sentences = [["the", "cat", "sat", "on", "the", "mat"], ["the", "dog", "sat", "on", "the", "rug"]]
model = Word2Vec(sentences, vector_size=100, window=5, sg=1, negative=5, min_count=1)
vec_cat = model.wv["cat"]                     # one fixed 100-d vector for "cat"
model.wv.most_similar("cat")                  # neighbors by cosine similarity

# Contextual: same word, two different vectors depending on sentence
from transformers import AutoTokenizer, AutoModel
import torch

tok = AutoTokenizer.from_pretrained("bert-base-uncased")
enc = AutoModel.from_pretrained("bert-base-uncased")

def token_vector(sentence, word):
    inputs = tok(sentence, return_tensors="pt")
    with torch.no_grad():
        hidden = enc(**inputs).last_hidden_state[0]     # [seq_len, hidden_dim]
    idx = tok.convert_ids_to_tokens(inputs["input_ids"][0]).index(word)
    return hidden[idx]

v1 = token_vector("I deposited money at the bank", "bank")
v2 = token_vector("I sat on the river bank", "bank")
# v1 and v2 are different vectors — same token ID, different context
```

## Common Mistakes

<ProsCons
  pros={[
    "Static embeddings are cheap to compute and store (a single lookup table), useful when latency or memory budgets rule out running a full Transformer forward pass.",
    "Contextual embeddings capture polysemy, negation, and long-range dependencies that static embeddings structurally cannot.",
    "Both families are trained without labeled data — the supervision signal comes entirely from raw text structure (the distributional hypothesis in action).",
  ]}
  cons={[
    "A common mistake: using a static embedding's nearest-neighbor list as evidence of 'true' meaning similarity — it reflects co-occurrence statistics, not semantics, so systematic corpus biases (including social biases) get baked directly into the vector geometry.",
    "Assuming contextual embeddings from a specific layer are 'the' representation to use for a downstream task — different Transformer layers capture different levels of abstraction (more syntactic in early layers, more semantic in later ones for many models), and the best layer to probe is task-dependent, not fixed.",
    "Comparing a static embedding's cosine similarity directly against a contextual model's token vector as if they're the same kind of object — contextual vectors are only meaningful in reference to the specific sentence they were computed from.",
  ]}
