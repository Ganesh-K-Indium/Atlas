## Intuition

The Bellman optimality equation tells us that $Q^*$ must satisfy a self-consistency condition: $Q^*(s,a)$ equals the immediate reward plus the discounted value of the best action in whatever state comes next. The catch is that this equation involves an expectation over the environment's transition distribution $P(s' \mid s, a)$ — and in most real problems, we don't know $P$. We can't compute the sum over $s'$ because we don't have the transition model.

Q-learning's insight is: we don't need the model if we can *sample* from it. Instead of computing the exact expectation, take one real transition $(s, a, r, s')$ the agent actually experienced, treat $r + \gamma \max_{a'} Q(s', a')$ as a noisy, one-sample estimate of what the Bellman equation says $Q(s,a)$ should equal, and nudge the current estimate a small step toward that target. Repeat forever, and — under suitable conditions — the estimate converges to $Q^*$.

<Callout type="tip" title="The one idea that matters">
  Q-learning is stochastic approximation applied to the Bellman optimality equation: replace the exact expectation over next states with a single sampled transition, and take a small step toward the resulting "TD target" instead of jumping there in one shot. Averaging over many noisy samples recovers the true expectation over time.
</Callout>

## Formal Definition

Recall the Bellman optimality equation for the action-value function:

$$
Q^*(s,a) = \mathbb{E}_{s' \sim P(\cdot \mid s,a)} \Big[ R(s,a,s') + \gamma \max_{a'} Q^*(s',a') \Big].
$$

Q-learning maintains a table (or function approximator) $Q(s,a)$ and updates it using experienced transitions $(s_t, a_t, r_{t+1}, s_{t+1})$ collected by acting in the environment, regardless of what policy generated those actions.

## Derivation: From Bellman Optimality to the TD Update

Define the **TD target** as a one-sample Monte Carlo estimate of the right-hand side of the Bellman optimality equation, using the single observed reward and next state in place of the full expectation over $P$:

$$
y_t = r_{t+1} + \gamma \max_{a'} Q(s_{t+1}, a').
$$

This is an unbiased estimate of $\mathbb{E}_{s'}[R + \gamma \max_{a'} Q^*(s', a')]$ *only if* $Q = Q^*$ already; while $Q$ is still being learned, $y_t$ is a biased "bootstrapped" estimate, since it uses the current (imperfect) $Q$ to estimate its own target. This bootstrapping is exactly what distinguishes **temporal-difference (TD) learning** from plain Monte Carlo estimation, which would instead wait for the full episode's actual return.

Define the **TD error** as the discrepancy between this target and the current estimate:

$$
\delta_t = y_t - Q(s_t, a_t) = r_{t+1} + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t).
$$

Stochastic approximation theory (Robbins–Monro) says: if you want to find the fixed point of an equation $Q = \mathcal{T}Q$ (here $\mathcal{T}$ is the Bellman optimality operator) but you can only observe noisy samples of $\mathcal{T}Q$, you can still converge to the fixed point by taking small steps in the direction of the sampled error, with a learning rate that decays appropriately:

$$
Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \, \delta_t = Q(s_t, a_t) + \alpha \Big[ r_{t+1} + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t) \Big].
$$

This is the **Q-learning update rule**. Each update moves $Q(s_t, a_t)$ a fraction $\alpha$ of the way toward the sampled Bellman target, exactly the way a running average moves a fraction of the way toward each new data point. Under standard conditions — every state-action pair visited infinitely often, and a learning rate schedule satisfying $\sum_t \alpha_t = \infty$, $\sum_t \alpha_t^2 < \infty$ — this procedure provably converges to $Q^*$ (Watkins & Dayan, 1992).

## On-Policy vs. Off-Policy

Look closely at the target: it uses $\max_{a'} Q(s_{t+1}, a')$ — the value of the *best* action in the next state, regardless of which action the agent actually takes next. This makes Q-learning **off-policy**: the policy used to select actions during learning (the "behavior policy," typically $\epsilon$-greedy, to ensure exploration) can be completely different from the policy whose value is being learned (the "target policy," which is always the greedy policy $\arg\max_a Q$).

Contrast this with **SARSA**, an *on-policy* TD control method whose update uses the action the agent *actually* takes next, $a_{t+1} \sim \pi$, instead of a max:

$$
Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \Big[ r_{t+1} + \gamma Q(s_{t+1}, a_{t+1}) - Q(s_t, a_t) \Big] \quad \text{(SARSA, on-policy)}.
$$

SARSA learns the value of the policy it's actually following (including its exploratory mistakes), so it tends to learn safer, more conservative behavior in risky environments; Q-learning learns the value of the optimal policy directly, even while behaving exploratorily, which can make it learn a more aggressive policy that occasionally suffers during exploration (the classic illustration is the "Cliff Walking" gridworld, where Q-learning learns the optimal but risky path along the cliff edge, while SARSA learns a safer, longer path).

## Exploration vs. Exploitation: $\epsilon$-Greedy

Q-learning's convergence guarantee requires every state-action pair to be visited infinitely often — but if the agent always acts greedily ($\arg\max_a Q$), it will quickly lock onto whatever action looks best early on and may never revisit alternatives that were actually better but under-sampled. This is the **exploration-exploitation tradeoff**: exploit current knowledge to get reward now, or explore to potentially discover better actions and improve future reward.

$\epsilon$-greedy is the simplest resolution: with probability $1-\epsilon$, take the greedy action $\arg\max_a Q(s,a)$ (exploitation); with probability $\epsilon$, take a uniformly random action (exploration). $\epsilon$ is typically annealed toward 0 over training, so the policy exploits more as $Q$ becomes more reliable.

## Code Example: Tabular Q-Learning

```python
import numpy as np

def q_learning_update(Q, s, a, r, s_next, alpha=0.1, gamma=0.99):
    """One Q-learning update step (tabular case).
    Q: array [num_states, num_actions]
    """
    td_target = r + gamma * np.max(Q[s_next])       # sampled Bellman-optimality target
    td_error = td_target - Q[s, a]
    Q[s, a] += alpha * td_error
    return Q, td_error

def epsilon_greedy(Q, s, epsilon, num_actions):
    if np.random.rand() < epsilon:
        return np.random.randint(num_actions)        # explore
    return np.argmax(Q[s])                            # exploit

def train_q_learning(env, num_states, num_actions, num_episodes=5000,
                      alpha=0.1, gamma=0.99, epsilon_start=1.0, epsilon_end=0.05):
    Q = np.zeros((num_states, num_actions))
    for ep in range(num_episodes):
        epsilon = max(epsilon_end, epsilon_start * (1 - ep / num_episodes))
        s = env.reset()
        done = False
        while not done:
            a = epsilon_greedy(Q, s, epsilon, num_actions)
            s_next, r, done, _ = env.step(a)
            Q, _ = q_learning_update(Q, s, a, r, s_next, alpha, gamma)
            s = s_next
    return Q
```

Note that the behavior policy (`epsilon_greedy`) and the target used in the update (`np.max(Q[s_next])`, i.e. the greedy policy) are different — this is off-policy learning made concrete in code.

## Common Mistakes

<Callout type="warning" title="Pitfalls to avoid">
  - **Setting ε to 0 too early.** If exploration stops before Q has converged, the agent can get stuck exploiting a suboptimal policy forever, since state-action pairs it never revisits can't be corrected.
  - **Confusing Q-learning's off-policy target with the behavior policy.** The max in the update is *not* the action the agent will actually take next — that's a common source of confusion with SARSA's update, which looks almost identical but uses the actually-taken next action.
  - **Ignoring maximization bias.** Because the target uses max_a' Q(s',a'), and Q is a noisy estimate, the max operator systematically overestimates true values (it's more likely to pick an action whose value is overestimated by noise than one that's underestimated). Double Q-learning addresses this by decoupling action selection from action evaluation using two separate Q-tables.
  - **Using a constant, too-large learning rate.** Without decay, Q-learning's updates can oscillate around the true value rather than converging tightly to it.
</Callout>
