## Intuition

Imagine fitting many different models to many different training sets, all drawn from the same underlying process. If your model is too simple (a straight line for curved data), it will consistently miss the pattern in the *same* systematic way no matter which training set you drew — that systematic error is **bias**. If your model is too flexible (a high-degree polynomial that snakes through every training point), it will fit the noise in each particular training set, so predictions swing wildly from one training set to the next — that instability is **variance**. Even a perfect model can't beat the **irreducible noise** already baked into the data-generating process.

<Callout type="tip" title="The core idea">
  Expected test error decomposes exactly into three non-negative, additive terms: $\text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$. Reducing model complexity typically lowers variance but raises bias, and vice versa — there is no free lunch, only a trade-off you can position along a complexity axis.
</Callout>

## Formal Definition

Assume the true data-generating process is

$$
y = f(x) + \varepsilon, \qquad \mathbb{E}[\varepsilon] = 0, \quad \mathrm{Var}(\varepsilon) = \sigma^2
$$

where $f$ is the (unknown) true function and $\varepsilon$ is irreducible noise, independent of $x$. We train an estimator $\hat{f}(x; D)$ on a training set $D$ drawn randomly from the data distribution. Because $D$ is random, $\hat{f}(x; D)$ is itself a random variable (for a fixed query point $x$) — different training sets yield different fitted functions.

We define:

- **Bias**: $\text{Bias}[\hat{f}(x)] = \mathbb{E}_D[\hat{f}(x; D)] - f(x)$ — how far the *average* prediction (averaged over all possible training sets) is from the truth.
- **Variance**: $\mathrm{Var}[\hat{f}(x)] = \mathbb{E}_D\big[(\hat{f}(x; D) - \mathbb{E}_D[\hat{f}(x; D)])^2\big]$ — how much the prediction fluctuates across different training sets.

## Derivation

We want the expected squared error at a fixed test point $x$, averaged over the noise $\varepsilon$ in the test label $y$ and over the randomness of the training set $D$:

$$
\mathbb{E}_{D,\varepsilon}\big[(y - \hat{f}(x; D))^2\big]
$$

**Step 1 — substitute $y = f(x) + \varepsilon$ and add/subtract $\mathbb{E}_D[\hat{f}(x;D)]$.** Let $\bar{f}(x) \equiv \mathbb{E}_D[\hat{f}(x;D)]$ denote the average prediction over training sets. Write:

$$
y - \hat{f}(x;D) = \big(f(x) + \varepsilon\big) - \hat{f}(x;D) = \underbrace{\big(f(x) - \bar f(x)\big)}_{-\text{Bias}} + \underbrace{\big(\bar f(x) - \hat f(x;D)\big)}_{-(\hat f - \bar f)} + \varepsilon
$$

**Step 2 — square and take expectation.** Squaring a sum of three terms produces three squared terms and three cross terms:

$$
\mathbb{E}\Big[\big((f - \bar f) + (\bar f - \hat f) + \varepsilon\big)^2\Big]
$$

**Step 3 — the cross terms vanish.**

- $\mathbb{E}_D[\bar f(x) - \hat f(x;D)] = \bar f(x) - \mathbb{E}_D[\hat f(x;D)] = 0$ by definition of $\bar f$, so any cross term multiplying $(\bar f - \hat f)$ by a constant (with respect to $D$) is zero in expectation.
- $\varepsilon$ has mean zero and is independent of $D$ and of $x$, so any cross term involving $\varepsilon$ vanishes in expectation (e.g. $\mathbb{E}_\varepsilon[\varepsilon] \cdot (\text{anything not depending on } \varepsilon) = 0$).

**Step 4 — what remains.**

$$
\mathbb{E}_{D,\varepsilon}\big[(y - \hat f(x;D))^2\big] = \underbrace{\big(f(x) - \bar f(x)\big)^2}_{\text{Bias}^2} + \underbrace{\mathbb{E}_D\big[(\hat f(x;D) - \bar f(x))^2\big]}_{\text{Variance}} + \underbrace{\mathbb{E}_\varepsilon[\varepsilon^2]}_{\sigma^2 \text{ (irreducible noise)}}
$$

$$
\boxed{\;\mathbb{E}\big[(y - \hat f(x))^2\big] = \text{Bias}[\hat f(x)]^2 + \mathrm{Var}[\hat f(x)] + \sigma^2\;}
$$

Every term on the right is non-negative, so this is a strict lower bound: no estimator can drive expected squared error below $\sigma^2$, no matter how good it is, because $\sigma^2$ reflects noise in $y$ itself that no function of $x$ can predict.

**Why complexity trades one for the other.** A high-capacity model (deep tree, high-degree polynomial, many parameters) can bend to fit whatever particular training set it sees, so $\hat f(x;D)$ tracks the training data's idiosyncrasies — $\bar f(x)$ (averaged over resamples) is close to $f(x)$ (low bias), but individual fits $\hat f(x;D)$ scatter widely around that average (high variance). A low-capacity model is too rigid to track training-set idiosyncrasies — individual fits barely change across resamples (low variance) — but its constrained shape can't match $f$ even on average (high bias).

## Code Example: Empirical Bias-Variance Decomposition via Bootstrap

We estimate bias and variance directly from data by resampling the training set many times, fitting a model on each bootstrap sample, and looking at how predictions at fixed test points vary.

```python
import numpy as np
from sklearn.tree import DecisionTreeRegressor

rng = np.random.default_rng(0)

# Ground truth: y = sin(x) + noise
def true_f(x):
    return np.sin(x)

def sample_data(n, noise_std=0.3):
    x = rng.uniform(0, 2 * np.pi, size=n)
    y = true_f(x) + rng.normal(0, noise_std, size=n)
    return x.reshape(-1, 1), y

x_test = np.linspace(0, 2 * np.pi, 50).reshape(-1, 1)
f_true = true_f(x_test.ravel())

n_bootstraps = 200
preds = np.zeros((n_bootstraps, len(x_test)))

for b in range(n_bootstraps):
    X_train, y_train = sample_data(n=80)
    model = DecisionTreeRegressor(max_depth=4, random_state=b)  # vary max_depth to see the trade-off
    model.fit(X_train, y_train)
    preds[b] = model.predict(x_test)

mean_pred = preds.mean(axis=0)
bias_sq = (mean_pred - f_true) ** 2
variance = preds.var(axis=0)
noise = 0.3 ** 2  # known here because we generated the data

total_error = bias_sq + variance + noise
print(f"Mean Bias^2:   {bias_sq.mean():.4f}")
print(f"Mean Variance: {variance.mean():.4f}")
print(f"Noise:         {noise:.4f}")
print(f"Sum:           {total_error.mean():.4f}")
```

Re-running this with `max_depth=1` (a stump) versus `max_depth=None` (fully grown tree) makes the trade-off visible: the stump has high average `bias_sq` and low `variance`; the fully grown tree has near-zero `bias_sq` but much larger `variance`.

## Common Mistakes

<Callout type="warning" title="Frequent errors">
  - **Confusing training error with the bias term.** Low training error does not imply low bias — an overfit model has near-zero bias on *average* (it can match $f$ closely in expectation over resamples) but that says nothing about how each individual fit deviates from the training data; training error near zero is actually the signature of high variance, not low bias.
  - **Treating the decomposition as if it applies to 0/1 loss the same way.** The clean additive decomposition above is exact for squared error. For classification with 0/1 loss, bias and variance can interact non-additively (variance can sometimes *reduce* error near a decision boundary) — see Kohavi & Wolpert (1996) and Domingos (2000) for the classification-specific decompositions.
  - **Assuming more data always requires more model complexity to avoid underfitting.** More data reduces variance for a fixed model, which often means you can now afford *more* complexity than a smaller dataset could support — but it does not by itself fix a biased model class (e.g., a linear model on inherently nonlinear data stays biased regardless of $n$).
  - **Forgetting that $\sigma^2$ is irreducible.** Teams sometimes keep tuning a model chasing test error improvements that are actually just noise variance in the test set itself, which no amount of modeling can eliminate.
</Callout>
