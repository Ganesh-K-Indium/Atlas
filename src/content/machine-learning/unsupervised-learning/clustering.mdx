## Intuition

Clustering asks a deceptively simple question: given a pile of unlabeled points, which ones "belong together"? There is no single correct answer — it depends entirely on what you mean by "together." K-Means means "close to the same center." DBSCAN means "connected through a dense region." Hierarchical clustering means "merges early under some linkage rule." Each definition carves the same data differently, and picking the wrong one for your data's shape is the single most common clustering mistake.

<Callout type="tip" title="One-sentence core idea">
  Clustering partitions points by a notion of similarity — K-Means groups by nearness to a centroid, DBSCAN groups by density connectivity, and hierarchical clustering builds a nested tree of merges — and each makes different assumptions about cluster shape that you must match to your data.
</Callout>

## Formal Definition

Given a dataset $X = \{x_1, \dots, x_n\} \subset \mathbb{R}^d$, clustering seeks an assignment of points to $k$ (or an unspecified number of) groups such that points in the same group are more similar to each other than to points in other groups. Unlike supervised classification, there is no ground-truth label to fit against — the objective *is* the similarity structure itself, which makes clustering fundamentally an optimization-over-structure problem rather than a prediction problem.

### K-Means: the within-cluster sum of squares

K-Means fixes the number of clusters $k$ in advance and represents each cluster $j$ by a centroid $\mu_j \in \mathbb{R}^d$. It seeks the assignment $c: \{1,\dots,n\} \to \{1,\dots,k\}$ and centroids that minimize the **within-cluster sum of squares (WCSS)**, also called inertia:

$$
J(c, \mu) = \sum_{i=1}^{n} \left\lVert x_i - \mu_{c(i)} \right\rVert^2 = \sum_{j=1}^{k} \sum_{i : c(i) = j} \left\lVert x_i - \mu_j \right\rVert^2
$$

This is a combinatorial optimization problem (assigning $n$ points to $k$ groups) coupled with a continuous one (placing $k$ centroids) — jointly minimizing both is NP-hard in general. In practice, we don't solve it exactly; we alternate between the two.

### DBSCAN: density-reachability

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) has no notion of a centroid at all. It defines clusters through two parameters: $\varepsilon$ (a neighborhood radius) and $\text{minPts}$ (a minimum neighborhood size), and three precise point categories:

- A point $p$ is a **core point** if its $\varepsilon$-neighborhood $N_\varepsilon(p) = \{q \in X : \lVert p - q \rVert \le \varepsilon\}$ contains at least $\text{minPts}$ points (including $p$ itself).
- A point $q$ is **directly density-reachable** from a core point $p$ if $q \in N_\varepsilon(p)$.
- $q$ is **density-reachable** from $p$ if there is a chain $p = p_0, p_1, \dots, p_m = q$ where each $p_{i+1}$ is directly density-reachable from $p_i$ (every intermediate point except possibly the last must be a core point).
- Two points $p, q$ are **density-connected** if there exists a point $o$ such that both $p$ and $q$ are density-reachable from $o$.

A **cluster** is a maximal set of points that are mutually density-connected. Any point that is not density-reachable from any core point is labeled **noise**. Non-core points that fall inside a core point's neighborhood but have too few neighbors of their own are called **border points** — they join a cluster but do not extend it further.

### Hierarchical (Agglomerative) Clustering: nested merges

Agglomerative clustering starts with every point as its own cluster and greedily merges the two closest clusters, repeated $n - 1$ times, producing a full binary merge tree (dendrogram). The result is not a single partition but a hierarchy — any horizontal cut of the dendrogram yields a valid clustering at that granularity.

The critical design choice is the **linkage criterion** — how to define distance between two *clusters* $A$ and $B$ (not points):

$$
d_{\text{single}}(A, B) = \min_{a \in A, b \in B} \lVert a - b \rVert \qquad \text{(nearest points — chains, sensitive to noise)}
$$

$$
d_{\text{complete}}(A, B) = \max_{a \in A, b \in B} \lVert a - b \rVert \qquad \text{(farthest points — compact, tight clusters)}
$$

$$
d_{\text{average}}(A, B) = \frac{1}{|A||B|} \sum_{a \in A} \sum_{b \in B} \lVert a - b \rVert \qquad \text{(mean pairwise distance — balanced)}
$$

$$
d_{\text{Ward}}(A, B) = \frac{|A||B|}{|A|+|B|} \left\lVert \mu_A - \mu_B \right\rVert^2 \qquad \text{(minimizes increase in total WCSS from merging)}
$$

Ward's method is special: it merges whichever pair of clusters causes the *smallest increase* in total within-cluster variance, making it directly analogous to the K-Means objective but applied greedily and hierarchically rather than via global reassignment.

## Mechanism / Derivation

### Lloyd's algorithm for K-Means

Since jointly minimizing $J(c, \mu)$ is intractable, Lloyd's algorithm performs **block coordinate descent**: fix one variable, optimize the other exactly, and alternate. Each half-step has a closed-form solution.

**Step 1 — Assignment step (fix $\mu$, optimize $c$).** For fixed centroids, each point's cost term $\lVert x_i - \mu_{c(i)} \rVert^2$ is minimized independently by assigning it to its nearest centroid:

$$
c(i) \leftarrow \arg\min_j \lVert x_i - \mu_j \rVert^2
$$

**Step 2 — Update step (fix $c$, optimize $\mu$).** For fixed assignments, $J$ is a separable sum of quadratics in each $\mu_j$. Setting the gradient to zero:

$$
\frac{\partial}{\partial \mu_j} \sum_{i:c(i)=j} \lVert x_i - \mu_j \rVert^2 = -2 \sum_{i:c(i)=j} (x_i - \mu_j) = 0 \implies \mu_j = \frac{1}{|\{i : c(i)=j\}|} \sum_{i:c(i)=j} x_i
$$

i.e., the optimal centroid is simply the mean of its assigned points (hence the name).

**Step 3 — Repeat** until assignments stop changing (or centroid movement falls below a tolerance).

Each of the two steps can only decrease or hold $J$ constant, and $J$ is bounded below by zero, so Lloyd's algorithm is **guaranteed to converge** — but only to a local optimum, since the overall problem is non-convex. This is why K-Means is run with multiple random initializations (`n_init`) and why smarter seeding (**K-Means++**, which picks initial centroids probabilistically weighted by squared distance to already-chosen centroids) substantially improves the odds of finding a good local optimum.

### DBSCAN's region-growing procedure

1. Pick an unvisited point $p$. Compute $N_\varepsilon(p)$.
2. If $|N_\varepsilon(p)| < \text{minPts}$, temporarily mark $p$ as noise (it may later become a border point of another cluster).
3. Otherwise, $p$ is a core point: start a new cluster and recursively absorb every point density-reachable from $p$ — for each neighbor that is itself a core point, add *its* neighborhood to the growing cluster as well.
4. Repeat with the next unvisited point until all points are processed.

Because clusters grow by chaining together overlapping dense neighborhoods rather than by distance to a single center, DBSCAN naturally discovers **arbitrarily shaped** clusters (e.g. two interleaved crescents) that K-Means — which always produces convex, roughly spherical (Voronoi) regions — cannot separate correctly.

## Code Example

```python
import numpy as np
from sklearn.cluster import KMeans, DBSCAN, AgglomerativeClustering
from sklearn.datasets import make_blobs

X, _ = make_blobs(n_samples=300, centers=4, cluster_std=0.8, random_state=42)

# K-Means
kmeans = KMeans(n_clusters=4, n_init=10, random_state=42).fit(X)
print("K-Means inertia:", kmeans.inertia_)

# DBSCAN
dbscan = DBSCAN(eps=0.6, min_samples=5).fit(X)
print("DBSCAN clusters found:", len(set(dbscan.labels_)) - (1 if -1 in dbscan.labels_ else 0))
print("Noise points:", (dbscan.labels_ == -1).sum())

# Agglomerative (Ward linkage)
agg = AgglomerativeClustering(n_clusters=4, linkage="ward").fit(X)
```

### Lloyd's algorithm from scratch (NumPy)

```python
import numpy as np

def kmeans_lloyd(X, k, n_iter=100, tol=1e-4, seed=0):
    rng = np.random.default_rng(seed)
    # K-Means++-lite: pick k distinct random points as initial centroids
    centroids = X[rng.choice(len(X), k, replace=False)]

    for _ in range(n_iter):
        # Assignment step: nearest centroid for every point
        dists = np.linalg.norm(X[:, None, :] - centroids[None, :, :], axis=2)  # (n, k)
        labels = np.argmin(dists, axis=1)

        # Update step: recompute centroids as cluster means
        new_centroids = np.array([
            X[labels == j].mean(axis=0) if np.any(labels == j) else centroids[j]
            for j in range(k)
        ])

        if np.linalg.norm(new_centroids - centroids) < tol:
            centroids = new_centroids
            break
        centroids = new_centroids

    inertia = np.sum((X - centroids[labels]) ** 2)
    return labels, centroids, inertia
```

## Complexity

| Method | Time complexity | Space complexity | Notes |
| --- | --- | --- | --- |
| K-Means (Lloyd's) | $O(n k d \cdot i)$ | $O(n d + k d)$ | $i$ = iterations to converge; scales well, but needs $k$ fixed upfront |
| DBSCAN | $O(n \log n)$ with spatial index (e.g. KD-tree), $O(n^2)$ naively | $O(n)$ | No $k$ needed; degrades in high dimensions where indexing loses effectiveness |
| Agglomerative (naive) | $O(n^3)$ | $O(n^2)$ (distance matrix) | $O(n^2 \log n)$ achievable with heaps for some linkages |

## Common Mistakes

- Running K-Means on unscaled features — a feature with a larger numeric range dominates the Euclidean distance and silently determines the clustering.
- Picking $k$ by eyeballing without validating using the elbow method (inertia vs. $k$) or silhouette score, then over-interpreting an arbitrary choice.
- Using DBSCAN with a single global `eps` on data where clusters have very different densities — sparse, real clusters get swallowed as noise while dense clusters over-merge.
- Assuming hierarchical clustering's dendrogram "height" is directly comparable across different linkage methods — single, complete, and average linkage produce dendrograms on different scales.
- Interpreting K-Means clusters as a discovered "ground truth" grouping rather than a byproduct of minimizing a specific, chosen objective — different objectives (e.g. density-based) can produce entirely different, equally valid answers.
