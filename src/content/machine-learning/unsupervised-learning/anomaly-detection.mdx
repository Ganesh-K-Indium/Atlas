## Intuition

Most anomaly detectors try to model what "normal" looks like and flag anything that doesn't fit. Isolation Forest inverts the question: instead of profiling normal points, it asks how *easy* a point is to isolate from the rest of the data using random cuts. Anomalies — being few and different — tend to sit in sparse regions, so a handful of random splits is usually enough to wall them off alone. Normal points, densely packed together, take many more splits to separate. That single observation — anomalies are "few and different," hence *easy to isolate* — is the entire algorithm.

<Callout type="tip" title="One-sentence core idea">
  Isolation Forest detects anomalies by measuring how few random splits it takes to isolate a point — short average path length means anomalous, long path length means normal — while density-based methods like Local Outlier Factor instead compare a point's local density to its neighbors' densities directly.
</Callout>

## Formal Definition / Theory

### Isolation Forest

An **isolation tree (iTree)** is built by recursively partitioning a dataset: at each node, pick a feature uniformly at random, then pick a split value uniformly at random between that feature's observed min and max, and split the data into two child nodes. Recursion stops when a node contains a single point (or all remaining points are identical) or a maximum tree height is reached.

The key statistic is the **path length** $h(x)$ of a point $x$: the number of edges traversed from the root to the leaf containing $x$. Because splits are random, anomalies — being isolated in sparse, low-density regions — are expected to require far fewer splits than points embedded in a dense cluster of similar points.

An **isolation forest** is an ensemble of $t$ such trees, each built on a random subsample of the data (subsampling both controls tree correlation and, empirically, improves anomaly detection by avoiding "swamping" and "masking" effects from large sample sizes). The anomaly score is based on the *average* path length $E[h(x)]$ across all trees.

Since path length in a random binary tree scales with the data size $n$ similarly to unsuccessful search depth in a Binary Search Tree, Liu, Ting & Zhou normalize using the average path length of an unsuccessful BST search:

$$
c(n) = 2H(n-1) - \frac{2(n-1)}{n}, \qquad H(i) \approx \ln(i) + 0.5772156649 \text{ (Euler-Mascheroni constant)}
$$

The **anomaly score** for a point $x$ is:

$$
s(x, n) = 2^{-\frac{E[h(x)]}{c(n)}}
$$

This formula is carefully chosen so its output is bounded in $(0, 1]$ and has an intuitive interpretation:

- $s \to 1$ as $E[h(x)] \to 0$ (isolated almost immediately) — **strong anomaly**.
- $s \to 0.5$ when $E[h(x)] \approx c(n)$ (average-case path length) — **no clear signal**, treat as normal.
- $s \to 0$ as $E[h(x)] \to n - 1$ (needs nearly every point to isolate it) — **strongly normal / deeply embedded**.

In practice, a threshold (commonly around 0.5–0.6, tuned via a contamination-rate hyperparameter) is applied to $s(x,n)$ to produce a binary anomaly label.

### Local Outlier Factor (LOF) — a density-based contrast

Isolation Forest is fundamentally a partitioning/tree-based method; LOF instead measures **local density deviation** directly, which matters because a purely global density threshold fails when different regions of the data have genuinely different densities (a point can be "locally anomalous" relative to a dense neighborhood while looking totally normal relative to the global average).

For a point $p$ with $k$-nearest neighbors, define the **reachability distance** from $p$ to neighbor $o$:

$$
\text{reach-dist}_k(p, o) = \max(\text{dist}_k(o), \, d(p, o))
$$

where $\text{dist}_k(o)$ is $o$'s own distance to *its* $k$-th nearest neighbor. This smooths out fluctuations when $p$ is very close to $o$.

The **local reachability density (LRD)** of $p$ is the inverse of the average reachability distance to its neighbors:

$$
\text{lrd}_k(p) = \left( \frac{\sum_{o \in N_k(p)} \text{reach-dist}_k(p, o)}{|N_k(p)|} \right)^{-1}
$$

Finally, the **LOF score** compares $p$'s density to the average density of its neighbors:

$$
\text{LOF}_k(p) = \frac{\sum_{o \in N_k(p)} \frac{\text{lrd}_k(o)}{\text{lrd}_k(p)}}{|N_k(p)|}
$$

$\text{LOF}(p) \approx 1$ means $p$'s density is comparable to its neighbors' (normal). $\text{LOF}(p) \gg 1$ means $p$ sits in a much sparser region than its neighbors do — a local outlier — even if that same absolute density would be unremarkable elsewhere in the dataset.

## Mechanism, step by step

**Building one isolation tree:**

1. Draw a random subsample of size $\psi$ (typically 256) from the training data.
2. At the root, select a feature $q$ uniformly at random among all features.
3. Select a split value $p$ uniformly at random in $[\min(q), \max(q)]$ over the current node's data.
3. Partition the data into "less than $p$" and "greater than or equal to $p$" child nodes.
4. Recurse on each child until a stopping condition (single point, identical points, or height limit $\approx \log_2 \psi$) is reached.

**Scoring a new point $x$:** drop $x$ down every tree in the forest, record the path length $h_i(x)$ in each tree, average across the $t$ trees to get $E[h(x)]$, then plug into $s(x,n) = 2^{-E[h(x)]/c(n)}$.

The elegance of this approach is that it needs **no distance computation at all** during tree construction (unlike LOF or k-NN-based methods, which require pairwise distances) — it only requires comparing a scalar feature value against a random threshold, which is why Isolation Forest scales to high-dimensional, large datasets much better than density- or distance-based competitors.

## Code Example

```python
import numpy as np
from sklearn.ensemble import IsolationForest
from sklearn.neighbors import LocalOutlierFactor
from sklearn.datasets import make_blobs

X, _ = make_blobs(n_samples=300, centers=1, cluster_std=1.0, random_state=42)
X = np.vstack([X, [[8, 8], [9, -9]]])  # inject two obvious outliers

# Isolation Forest
iso = IsolationForest(n_estimators=100, contamination=0.02, random_state=42)
iso_labels = iso.fit_predict(X)          # -1 = anomaly, 1 = normal
iso_scores = iso.decision_function(X)    # higher = more normal

# Local Outlier Factor (unsupervised, no separate .predict on new data by default)
lof = LocalOutlierFactor(n_neighbors=20, contamination=0.02)
lof_labels = lof.fit_predict(X)          # -1 = anomaly, 1 = normal
lof_scores = lof.negative_outlier_factor_
```

### Isolation score computation from scratch (NumPy)

```python
import numpy as np

def build_itree(X, height=0, height_limit=8):
    if len(X) <= 1 or height >= height_limit:
        return {"size": len(X)}
    feature = np.random.randint(X.shape[1])
    col = X[:, feature]
    if col.min() == col.max():
        return {"size": len(X)}
    split = np.random.uniform(col.min(), col.max())
    left, right = X[col < split], X[col >= split]
    return {
        "feature": feature, "split": split,
        "left": build_itree(left, height + 1, height_limit),
        "right": build_itree(right, height + 1, height_limit),
    }

def path_length(x, tree, height=0):
    if "feature" not in tree:  # leaf
        n = tree["size"]
        c_n = 2 * (np.log(n - 1) + 0.5772156649) - 2 * (n - 1) / n if n > 1 else 0
        return height + c_n
    branch = tree["left"] if x[tree["feature"]] < tree["split"] else tree["right"]
    return path_length(x, branch, height + 1)

def anomaly_score(x, forest, n):
    avg_h = np.mean([path_length(x, tree) for tree in forest])
    c_n = 2 * (np.log(n - 1) + 0.5772156649) - 2 * (n - 1) / n
    return 2 ** (-avg_h / c_n)
```

## Complexity

| Method | Time complexity (train) | Time complexity (score) | Space complexity | Notes |
| --- | --- | --- | --- | --- |
| Isolation Forest | $O(t \cdot \psi \log \psi)$ | $O(t \log \psi)$ per point | $O(t \psi)$ | $t$ = trees, $\psi$ = subsample size; independent of full dataset size $n$ at train time |
| Local Outlier Factor | $O(n \log n)$ with spatial index, $O(n^2)$ naive | $O(k \log n)$ per new point (with index) | $O(n)$ | Requires k-NN search; degrades in high dimensions |
| One-Class SVM (statistical baseline) | $O(n^2)$ to $O(n^3)$ | $O(n_{SV} \cdot d)$ | $O(n^2)$ | Kernel-based; scales poorly to large n |

## Common Mistakes

- Treating the Isolation Forest anomaly score as a calibrated probability of being anomalous — it is a relative, ensemble-averaged path-length statistic, not a probability; only the *threshold* you choose (often via a `contamination` hyperparameter) turns it into a hard decision.
- Using LOF or other distance-based detectors on high-dimensional data without dimensionality reduction first — as dimensionality grows, pairwise distances concentrate (everything becomes roughly equidistant), destroying the density contrast LOF relies on.
- Forgetting that unsupervised anomaly detectors have no ground truth to validate against during training — evaluation requires either held-out labeled anomalies or domain-expert review, and the `contamination` parameter is a strong, easily-misset prior.
- Applying a single global density/distance threshold to data with naturally varying density regions — this is exactly the failure mode LOF was designed to avoid, but reappears if you swap in a simpler global method.
- Not re-scaling features before Isolation Forest — while it's more robust to scale than distance-based methods (since splits are per-feature, not on combined distance), extremely skewed feature ranges can still bias which features get picked and where random thresholds land.
