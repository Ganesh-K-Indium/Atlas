## Intuition

A single decision tree is a weak model — shallow trees underfit, deep trees overfit. **Boosting** builds a *sequence* of shallow trees, where each new tree is trained specifically to correct the errors of everything built so far. Instead of one strong model, you get an additive ensemble of many weak ones, each nudging the prediction a little closer to the truth.

<Callout type="tip" title="The core idea">
  Gradient boosting fits each new tree to the **negative gradient of the loss** with respect to the current ensemble's predictions — i.e., each tree approximates "which direction, and by how much, should I move each prediction to reduce the loss." This is functional gradient descent, where the "parameter" being updated is the whole function (the ensemble), not a fixed-size weight vector.
</Callout>

## Formal Derivation

We build an additive model incrementally:

$$
F_m(x) = F_{m-1}(x) + \eta \, h_m(x)
$$

where $F_{m-1}$ is the ensemble after $m-1$ rounds, $h_m$ is the new tree, and $\eta$ is the learning rate (shrinkage). We want to choose $h_m$ to minimize the loss $\mathcal{L} = \sum_i \ell(y_i, F_{m-1}(x_i) + h_m(x_i))$.

**First-order (gradient boosting, Friedman 2001):** Taylor-expand $\ell$ around $F_{m-1}(x_i)$ to first order and fit $h_m$ to the negative gradient (pseudo-residual):

$$
r_i^{(m)} = -\left[ \frac{\partial \ell(y_i, F(x_i))}{\partial F(x_i)} \right]_{F = F_{m-1}}
$$

The new tree $h_m$ is trained by ordinary least-squares regression against these pseudo-residuals $r_i^{(m)}$ — literally: "fit a tree that predicts the direction each prediction needs to move."

**Second-order (XGBoost's innovation, Chen & Guestrin 2016):** Instead of a first-order Taylor expansion, XGBoost expands to **second order**, using both gradient $g_i$ and Hessian $h_i$ of the loss:

$$
\mathcal{L}^{(m)} \approx \sum_i \left[ g_i \, h_m(x_i) + \tfrac{1}{2} h_i \, h_m(x_i)^2 \right] + \Omega(h_m)
$$

where $\Omega(h_m) = \gamma T + \tfrac{1}{2}\lambda \sum_j w_j^2$ is an explicit regularization term penalizing the number of leaves $T$ and the leaf weights $w_j$ — this regularization is baked directly into the objective, not bolted on afterward. For a fixed tree structure, the optimal leaf weight has a closed form:

$$
w_j^* = -\frac{\sum_{i \in I_j} g_i}{\sum_{i \in I_j} h_i + \lambda}
$$

which is why XGBoost can evaluate candidate splits extremely efficiently — the optimal leaf value and the resulting loss reduction are both closed-form given the split.

## How the Three Libraries Diverge

All three implement gradient-boosted trees; they differ in *how* they grow trees and handle features.

<ComparisonTable
  headers={["Library", "Tree growth", "Key innovation", "Best at"]}
  rows={[
    ["XGBoost", "Level-wise (depth-first, balanced)", "Second-order (Newton) boosting + explicit L1/L2 regularization on leaf weights", "General-purpose robustness; the long-standing competition baseline"],
    ["LightGBM", "Leaf-wise (grows the leaf with max loss reduction first)", "Histogram-based binning + Gradient-based One-Side Sampling (GOSS) + Exclusive Feature Bundling (EFB)", "Very large datasets and high-dimensional sparse features; fastest training"],
    ["CatBoost", "Symmetric (oblivious) trees — same split condition across a whole level", "Ordered boosting + native target statistics for categorical features (avoids target leakage)", "Datasets with many high-cardinality categorical features; minimal tuning needed"],
  ]}
/>

**Why leaf-wise (LightGBM) is faster but riskier:** growing the single leaf with the largest loss reduction, rather than expanding every leaf at the current depth, converges to lower loss with fewer splits — but can produce deep, unbalanced trees that overfit small datasets if `max_depth` isn't capped.

**Why CatBoost's ordered boosting matters:** naively computing target statistics for categorical features (e.g., "mean target value for this category") using the *same* data point being predicted causes target leakage — the model implicitly sees its own label. Ordered boosting computes these statistics using only data that would have been "seen before" the current example in a random permutation, closing that leak.

## Minimal Example

```python
import xgboost as xgb
import lightgbm as lgb
from catboost import CatBoostClassifier

# XGBoost — second-order boosting with explicit regularization
xgb_model = xgb.XGBClassifier(
    n_estimators=500, max_depth=6, learning_rate=0.05,
    reg_lambda=1.0, reg_alpha=0.0, tree_method="hist",
)

# LightGBM — leaf-wise growth, histogram binning
lgb_model = lgb.LGBMClassifier(
    n_estimators=500, num_leaves=31, learning_rate=0.05,
)

# CatBoost — native categorical handling, no manual encoding needed
cat_model = CatBoostClassifier(
    iterations=500, depth=6, learning_rate=0.05,
    cat_features=["country", "device_type"], verbose=False,
)
```

## Complexity

| Operation | Complexity | Notes |
| --- | --- | --- |
| Exact greedy split finding | $O(n \cdot d \cdot \log n)$ per tree | $n$ samples, $d$ features; must sort every feature |
| Histogram-based split finding (XGBoost `hist`, LightGBM default) | $O(n \cdot d)$ per tree, plus $O(\text{bins})$ per split search | Bucket continuous features into ~256 bins once, reused across trees |
| Prediction (single tree) | $O(\text{depth})$ | Trees are typically shallow (6–10 levels) |
