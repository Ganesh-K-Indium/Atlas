## Intuition

To classify an email as spam or not spam, you'd ideally want $P(\text{spam} \mid \text{every word and its position and its interactions with every other word})$ — but estimating that joint distribution from a finite dataset is hopeless; there are exponentially many possible word combinations. Naive Bayes sidesteps this by simply *pretending* that, given the class, every feature is independent of every other feature. That assumption is almost always false in reality (word choice absolutely is correlated), which is exactly why the model is called "naive" — yet the classifier that results from this simplification is fast, needs very little data, and is a genuinely strong baseline for text and other high-dimensional sparse data.

<Callout type="tip" title="The core idea">
  Naive Bayes converts a hard joint-probability estimation problem into $d$ easy univariate ones by assuming features are conditionally independent given the class. It then picks whichever class maximizes the product of those individual per-feature likelihoods, weighted by the class's prior probability.
</Callout>

## Formal Definition

Given a feature vector $x = (x_1, \dots, x_d)$ and class label $y \in \{1, \dots, K\}$, Bayes' theorem gives the posterior class probability as:

$$
P(y \mid x) = \frac{P(x \mid y)\,P(y)}{P(x)}
$$

Since $P(x)$ is the same normalizing constant regardless of which class we're scoring, classification reduces to:

$$
\hat{y} = \arg\max_y \; P(x \mid y)\,P(y)
$$

The entire difficulty is in $P(x \mid y) = P(x_1, \dots, x_d \mid y)$ — the joint likelihood of all $d$ features given the class, which in general requires modeling every feature interaction.

## Derivation: from Bayes' theorem to the naive classifier

By the chain rule of probability, the joint likelihood can always be factored exactly (no assumption yet) as:

$$
P(x_1, \dots, x_d \mid y) = P(x_1 \mid y)\, P(x_2 \mid x_1, y)\, P(x_3 \mid x_1, x_2, y)\cdots P(x_d \mid x_1,\dots,x_{d-1}, y)
$$

This is exact but useless in practice — the later terms condition on an ever-growing combination of prior features, and we would need to have observed enough data to estimate every such combination. This is where the **"naive" conditional independence assumption** is introduced explicitly:

$$
\text{Assume: } \; x_i \perp x_j \mid y \quad \text{for all } i \neq j
$$

i.e., once you know the class, knowing the value of one feature tells you nothing about any other feature's value. Under this assumption every conditioning term above collapses to just $P(x_i \mid y)$:

$$
P(x_1, \dots, x_d \mid y) \approx \prod_{i=1}^d P(x_i \mid y)
$$

Substituting back into the Bayes' rule decision rule gives the **Naive Bayes classifier**:

$$
\hat{y} = \arg\max_y \; P(y) \prod_{i=1}^d P(x_i \mid y)
$$

In practice this product is computed in log space to avoid numerical underflow from multiplying many small probabilities:

$$
\hat{y} = \arg\max_y \; \left[ \log P(y) + \sum_{i=1}^d \log P(x_i \mid y) \right]
$$

Note this is now a **linear function of the log-likelihoods**, which is why Naive Bayes decision boundaries are linear for many common choices of $P(x_i \mid y)$ (e.g., when features are Bernoulli or when they're Gaussian with shared variance across classes).

**Why "naive" and why it works anyway:** the independence assumption is almost never true — in a spam email, the words "free" and "money" absolutely tend to co-occur, so treating them as independent double-counts (or under-counts) evidence. But for *classification* what matters is not the accuracy of the estimated probabilities themselves but whether $\arg\max_y$ picks the correct class — Naive Bayes probabilities are frequently badly calibrated (miscalibrated near 0 or 1) even when the resulting *decisions* are correct, because the ranking across classes is often preserved even under the wrong independence assumption. It also needs far fewer training examples than a full joint model since it estimates $d$ one-dimensional distributions rather than one $d$-dimensional distribution.

### Variants: how $P(x_i \mid y)$ is modeled

- **Multinomial NB**: $x_i$ is a count (e.g., word frequency in a document); $P(x_i \mid y)$ follows a multinomial distribution — the standard choice for bag-of-words text classification.
- **Bernoulli NB**: $x_i \in \{0, 1\}$ indicates feature presence/absence, ignoring counts.
- **Gaussian NB**: continuous $x_i$ is modeled as $x_i \mid y \sim \mathcal{N}(\mu_{y,i}, \sigma_{y,i}^2)$, estimated per class per feature from the training data.

### Laplace (additive) smoothing

If a word never appears in training documents of a given class, its estimated likelihood $P(x_i \mid y) = 0$, which zeroes out the *entire* product for that class regardless of all other evidence — a single unseen word can veto an otherwise overwhelming case. Laplace smoothing fixes this by adding a pseudocount $\alpha$ (typically 1) to every count:

$$
P(x_i \mid y) = \frac{\text{count}(x_i, y) + \alpha}{\text{count}(y) + \alpha \cdot |V|}
$$

where $|V|$ is the vocabulary size, ensuring every probability stays strictly positive.

## Code Example

```python
import numpy as np
from sklearn.naive_bayes import MultinomialNB, GaussianNB
from sklearn.feature_extraction.text import CountVectorizer

# --- Text classification with Multinomial Naive Bayes ---
docs = ["free money now", "meeting agenda attached", "free free free win money",
        "project deadline tomorrow", "win a free prize", "quarterly report attached"]
labels = [1, 0, 1, 0, 1, 0]  # 1 = spam, 0 = not spam

vectorizer = CountVectorizer()
X = vectorizer.fit_transform(docs)
clf = MultinomialNB(alpha=1.0)  # alpha = Laplace smoothing pseudocount
clf.fit(X, labels)
print(clf.predict(vectorizer.transform(["free free win"])))

# --- From-scratch Multinomial Naive Bayes with Laplace smoothing ---
def fit_naive_bayes(X, y, alpha=1.0):
    classes = np.unique(y)
    class_priors = {c: np.mean(y == c) for c in classes}
    feature_probs = {}
    for c in classes:
        X_c = X[y == c]
        counts = X_c.sum(axis=0) + alpha           # additive smoothing
        feature_probs[c] = counts / counts.sum()
    return class_priors, feature_probs

def predict_naive_bayes(X, class_priors, feature_probs):
    scores = []
    for x in X:
        class_scores = {
            c: np.log(class_priors[c]) + np.sum(x * np.log(feature_probs[c]))
            for c in class_priors
        }
        scores.append(max(class_scores, key=class_scores.get))
    return scores

X_dense = X.toarray()
priors, probs = fit_naive_bayes(X_dense, np.array(labels))
print(predict_naive_bayes(X_dense, priors, probs))
```

## Common Mistakes

<Callout type="warning" title="Frequent errors">
  - Forgetting Laplace/additive smoothing, causing unseen words at test time to zero out an entire class's score.
  - Using Gaussian NB on features that are clearly not unimodal/bell-shaped per class — check the assumption or bin/transform the feature first.
  - Treating Naive Bayes's output probabilities as well-calibrated — they are often overconfident (pushed toward 0 or 1) due to the independence assumption compounding correlated evidence.
  - Mixing count-based features with Multinomial NB's Bernoulli-style presence features without picking the matching variant.
</Callout>
