## Intuition

Every ML model that outputs a confidence score, every A/B test, every loss function derived from "maximum likelihood" — all of it rests on a small set of rules about how to quantify and combine uncertainty. Probability theory is that rule set. It doesn't tell you *why* something is uncertain (measurement noise? inherent randomness? your own ignorance?) — it just gives you consistent arithmetic for reasoning about degrees of belief or frequencies, whatever your interpretation.

<Callout type="tip" title="One-sentence definition">
  Probability is a function that assigns a number in $[0, 1]$ to events, satisfying three simple axioms — everything else (conditioning, Bayes' theorem, independence) is derived from those three rules alone.
</Callout>

The single most important derived rule is **conditioning**: updating your probabilities once you learn something. Almost every practical use of probability in ML — classification, filtering, ranking — is a conditioning computation in disguise: "given this input, what's the probability of that label?"

## Formal Definition: The Kolmogorov Axioms

A **probability space** is a triple $(\Omega, \mathcal{F}, P)$:

- $\Omega$ — the **sample space**: the set of all possible outcomes.
- $\mathcal{F}$ — a collection of subsets of $\Omega$ called **events** (technically a $\sigma$-algebra, so it's closed under complement and countable union).
- $P: \mathcal{F} \to [0, 1]$ — the **probability measure**, satisfying:

$$
\begin{aligned}
&\textbf{1. Non-negativity: } P(A) \geq 0 \text{ for all } A \in \mathcal{F} \\
&\textbf{2. Normalization: } P(\Omega) = 1 \\
&\textbf{3. Countable additivity: } P\left(\bigcup_{i=1}^{\infty} A_i\right) = \sum_{i=1}^{\infty} P(A_i) \text{ for pairwise disjoint } A_1, A_2, \ldots
\end{aligned}
$$

Everything else — that $P(\emptyset) = 0$, that $P(A^c) = 1 - P(A)$, that $P(A \cup B) = P(A) + P(B) - P(A \cap B)$ — is a theorem provable from just these three axioms, not an extra assumption.

**Proof sketch that $P(A^c) = 1 - P(A)$:** $A$ and $A^c$ are disjoint and $A \cup A^c = \Omega$. By axiom 3, $P(A) + P(A^c) = P(\Omega) = 1$ (axiom 2), so $P(A^c) = 1 - P(A)$.

## Conditional Probability and Independence

The **conditional probability** of $A$ given $B$ (with $P(B) > 0$) is defined as:

$$
P(A \mid B) = \frac{P(A \cap B)}{P(B)}
$$

This is a *definition*, not a derived fact — but it's the only definition consistent with the intuitive idea of "restricting the sample space to $B$ and renormalizing." If you condition on $B$, the new universe is $B$ itself, so probabilities must be rescaled by $P(B)$ to sum to 1 again.

Two events are **independent** if learning one tells you nothing about the other:

$$
P(A \mid B) = P(A) \quad \iff \quad P(A \cap B) = P(A) P(B)
$$

The right-hand form is preferred as the formal definition because it doesn't require $P(B) > 0$ and it's symmetric in $A$ and $B$ by inspection.

<Callout type="warning" title="Independence ≠ mutually exclusive">
  These are frequently confused. Mutually exclusive events ($A \cap B = \emptyset$) are *maximally dependent* — if $A$ happens, $B$ provably didn't. Independent events satisfy $P(A \cap B) = P(A)P(B) > 0$ whenever both have positive probability, so they can co-occur.
</Callout>

## Derivation: Bayes' Theorem from the Definition of Conditioning

Bayes' theorem is not a separate axiom — it falls straight out of the symmetry of the joint probability $P(A \cap B)$.

**Step 1.** From the definition of conditional probability, in both directions:

$$
P(A \mid B) \, P(B) = P(A \cap B) = P(B \mid A) \, P(A)
$$

**Step 2.** Divide both sides by $P(B)$ (assuming $P(B) > 0$):

$$
P(A \mid B) = \frac{P(B \mid A) \, P(A)}{P(B)}
$$

**Step 3.** Expand $P(B)$ using the **law of total probability** — if $A_1, \ldots, A_n$ partition $\Omega$:

$$
P(B) = \sum_{i=1}^n P(B \mid A_i) P(A_i)
$$

Substituting gives the full form used in ML (e.g. Naive Bayes classifiers, spam filters, medical diagnosis):

$$
P(A_k \mid B) = \frac{P(B \mid A_k) \, P(A_k)}{\sum_{i=1}^n P(B \mid A_i) P(A_i)}
$$

**Worked numeric example.** A rare disease affects 1% of a population ($P(D) = 0.01$). A test has 95% sensitivity ($P(+ \mid D) = 0.95$) and 90% specificity ($P(- \mid D^c) = 0.90$, so $P(+ \mid D^c) = 0.10$). Given a positive test, what's $P(D \mid +)$?

$$
P(D \mid +) = \frac{P(+ \mid D) P(D)}{P(+ \mid D)P(D) + P(+ \mid D^c)P(D^c)} = \frac{0.95 \times 0.01}{0.95 \times 0.01 + 0.10 \times 0.99} = \frac{0.0095}{0.1085} \approx 0.0876
$$

Only about **8.8%**, despite a seemingly accurate test — because the disease is rare, false positives from the large healthy population outnumber true positives. This is the base-rate fallacy, and it's the same arithmetic that governs precision in imbalanced classification problems.

## The Chain Rule and Its Use in Generative Models

For any events (or random variables) $X_1, \ldots, X_n$, repeatedly applying the definition of conditional probability gives the **chain rule of probability**:

$$
P(X_1, X_2, \ldots, X_n) = P(X_1) \prod_{i=2}^n P(X_i \mid X_1, \ldots, X_{i-1})
$$

This is *exactly* how autoregressive language models factorize the probability of a token sequence: $P(x_1, \ldots, x_T) = \prod_t P(x_t \mid x_{<t})$. Every next-token prediction is one factor in this product — see [The Transformer Architecture](/docs/architectures/transformers/transformer-architecture) (roadmap) for how this factorization is implemented.

## Code Example

```python
import numpy as np

# Simulating Bayes' theorem: verify the disease/test example via Monte Carlo
rng = np.random.default_rng(0)
n = 10_000_000

has_disease = rng.random(n) < 0.01                     # P(D) = 0.01
test_positive = np.where(
    has_disease,
    rng.random(n) < 0.95,                               # sensitivity
    rng.random(n) < 0.10,                                # false positive rate
)

p_disease_given_positive = has_disease[test_positive].mean()
print(f"P(D | +) ≈ {p_disease_given_positive:.4f}")     # ≈ 0.0876, matches closed form
```
