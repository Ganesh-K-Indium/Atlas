## Intuition

A matrix looks like a static grid of numbers, but the useful way to think about it is as a **function**: given a matrix $A \in \mathbb{R}^{m \times n}$, the map $x \mapsto Ax$ takes vectors in $\mathbb{R}^n$ and produces vectors in $\mathbb{R}^m$. Crucially, this map is *linear* — it never bends space, only stretches, rotates, reflects, or projects it. Every layer of a neural network before its nonlinearity ($Wx + b$) is exactly this kind of map, which is why matrix operations are the computational core of deep learning.

<Callout type="tip" title="One-sentence definition">
  A matrix is a rectangular array of numbers that represents a linear transformation between vector spaces; matrix multiplication is function composition written as arithmetic.
</Callout>

## Formal Definition

An $m \times n$ matrix $A$ is an array with $m$ rows and $n$ columns, $A \in \mathbb{R}^{m \times n}$, with entries $A_{ij}$ for $i = 1, \ldots, m$ and $j = 1, \ldots, n$.

### Core Operations

**Addition** (same shape only): $(A + B)_{ij} = A_{ij} + B_{ij}$.

**Scalar multiplication:** $(cA)_{ij} = c\, A_{ij}$.

**Matrix–vector product:** for $A \in \mathbb{R}^{m \times n}$ and $x \in \mathbb{R}^n$,

$$
(Ax)_i = \sum_{j=1}^n A_{ij} x_j
$$

— i.e., each output entry is the dot product of a row of $A$ with $x$. Equivalently, $Ax$ is a *linear combination of the columns of $A$*, weighted by the entries of $x$ — a viewpoint that turns out to be the more useful one for understanding rank, span, and the column space.

**Matrix–matrix product:** for $A \in \mathbb{R}^{m \times n}$ and $B \in \mathbb{R}^{n \times p}$,

$$
(AB)_{ik} = \sum_{j=1}^n A_{ij} B_{jk}, \qquad AB \in \mathbb{R}^{m \times p}
$$

The inner dimensions must match ($n = n$); the result takes the outer dimensions ($m \times p$). $AB$ represents applying the transformation $B$ first, then $A$ — matrix multiplication *is* function composition: $(AB)x = A(Bx)$.

**Transpose:** $(A^\top)_{ij} = A_{ji}$, flipping rows and columns. Key identity: $(AB)^\top = B^\top A^\top$ (the order reverses).

**Identity and inverse:** $I_n \in \mathbb{R}^{n \times n}$ has 1s on the diagonal and 0s elsewhere, and satisfies $IA = AI = A$. A square matrix $A$ is **invertible** if there exists $A^{-1}$ with $AA^{-1} = A^{-1}A = I$; this exists iff $\det(A) \neq 0$, equivalently iff $A$ has full rank.

### Properties That Trip People Up

- Matrix multiplication is **associative** ($(AB)C = A(BC)$) and **distributive** ($A(B+C) = AB + AC$), but **not commutative** in general: $AB \neq BA$ even when both products are defined and same-shaped.
- $(AB)^{-1} = B^{-1}A^{-1}$ (order reverses, same as transpose).
- The **rank** of $A$ is the dimension of its column space (equivalently, row space) — the number of linearly independent columns. $\text{rank}(A) \le \min(m, n)$; if equality holds, $A$ has **full rank**.

## Worked Example: Composing Two Transformations

Let $A$ rotate vectors in $\mathbb{R}^2$ by $90°$ counterclockwise, and $B$ scale the $x$-axis by 2:

$$
A = \begin{pmatrix} 0 & -1 \\ 1 & 0 \end{pmatrix}, \qquad
B = \begin{pmatrix} 2 & 0 \\ 0 & 1 \end{pmatrix}
$$

**Step 1 — compute $AB$ (scale, then rotate):**

$$
AB = \begin{pmatrix} 0 & -1 \\ 1 & 0 \end{pmatrix}\begin{pmatrix} 2 & 0 \\ 0 & 1 \end{pmatrix}
= \begin{pmatrix} 0\cdot2+(-1)\cdot0 & 0\cdot0+(-1)\cdot1 \\ 1\cdot2+0\cdot0 & 1\cdot0+0\cdot1 \end{pmatrix}
= \begin{pmatrix} 0 & -1 \\ 2 & 0 \end{pmatrix}
$$

**Step 2 — compute $BA$ (rotate, then scale):**

$$
BA = \begin{pmatrix} 2 & 0 \\ 0 & 1 \end{pmatrix}\begin{pmatrix} 0 & -1 \\ 1 & 0 \end{pmatrix}
= \begin{pmatrix} 2\cdot0+0\cdot1 & 2\cdot(-1)+0\cdot0 \\ 0\cdot0+1\cdot1 & 0\cdot(-1)+1\cdot0 \end{pmatrix}
= \begin{pmatrix} 0 & -2 \\ 1 & 0 \end{pmatrix}
$$

**Conclusion:** $AB \neq BA$ — this confirms matrix multiplication is not commutative, and makes the point concrete: rotating a stretched square gives a different shape than stretching a rotated square. Order of operations matters, exactly as it does when stacking layers in a network.

**Step 3 — apply to a vector.** Take $x = (1, 0)^\top$. Then $Bx = (2, 0)^\top$ (stretched), and $A(Bx) = (0, 2)^\top$ (then rotated 90°). Directly: $(AB)x = \begin{pmatrix} 0 & -1 \\ 2 & 0 \end{pmatrix}\begin{pmatrix}1\\0\end{pmatrix} = \begin{pmatrix}0\\2\end{pmatrix}$ ✓ — matches, confirming $(AB)x = A(Bx)$.

## Code Example

```python
import numpy as np

A = np.array([[0, -1],
              [1,  0]])   # 90-degree rotation
B = np.array([[2, 0],
              [0, 1]])    # scale x-axis by 2

AB = A @ B
BA = B @ A
print(AB)   # [[ 0 -1] [ 2  0]]
print(BA)   # [[ 0 -2] [ 1  0]]
print(np.array_equal(AB, BA))  # False -> non-commutative

x = np.array([1, 0])
print((A @ B) @ x, A @ (B @ x))  # both -> [0 2], confirming associativity

rank = np.linalg.matrix_rank(A)
inv_A = np.linalg.inv(A)
print(rank, inv_A)
```

## Complexity

| Operation | Time complexity | Space |
| --- | --- | --- |
| Matrix addition ($m \times n$) | $O(mn)$ | $O(mn)$ |
| Matrix–vector product ($m \times n$ by $n$) | $O(mn)$ | $O(m)$ |
| Matrix–matrix product (naive, $m \times n$ by $n \times p$) | $O(mnp)$ | $O(mp)$ |
| Matrix–matrix product (Strassen-like) | $O(n^{2.807})$ (square, asymptotic; rarely used in practice) | $O(n^2)$ |
| Matrix inverse (Gaussian elimination / LU) | $O(n^3)$ | $O(n^2)$ |
| Rank (via row reduction) | $O(mn \cdot \min(m,n))$ | $O(mn)$ |

## Common Mistakes

<ProsCons
  pros={[
    "Thinking of Ax as 'a combination of A's columns' rather than 'row-by-row dot products' — this view makes column space, rank, and solvability of Ax=b far more intuitive.",
    "Remembering that (AB)^T = B^T A^T and (AB)^-1 = B^-1 A^-1 — the reversal pattern shows up constantly in backpropagation derivations.",
  ]}
  cons={[
    "Assuming AB = BA — true only in special cases (e.g., A and B share eigenvectors, or one is the identity/a scalar multiple of it).",
    "Forgetting that matrix inversion requires a square, full-rank matrix — non-square or rank-deficient matrices only have a pseudoinverse (see Moore-Penrose), not a true inverse.",
    "Using naive O(n^3) matrix multiplication in a hot loop when a batched, vectorized BLAS call (as in NumPy/PyTorch) is often 10-100x faster due to cache-friendly memory access and hardware-level optimization.",
  ]}
