## Intuition

Most vectors change *direction* when you apply a matrix to them. An eigenvector is one of the special directions a matrix leaves alone — it only gets scaled, never rotated or reflected. The scale factor is the eigenvalue.

<Callout type="tip" title="One-sentence definition">
  $v$ is an eigenvector of $A$ with eigenvalue $\lambda$ if $Av$ points along the same line as $v$: $Av = \lambda v$.
</Callout>

Picture a matrix as a transformation of the plane — it stretches space along certain axes and compresses it along others. Eigenvectors are exactly those axes. If you decompose *any* vector into components along the eigenvectors, applying the matrix repeatedly just scales each component independently. That's what makes eigen-decomposition so useful: it turns a coupled, hard-to-reason-about linear system into a set of independent 1-D scalings.

## Formal Definition

For a square matrix $A \in \mathbb{R}^{n \times n}$, a nonzero vector $v \in \mathbb{R}^n$ is an eigenvector with eigenvalue $\lambda \in \mathbb{C}$ if:

$$
Av = \lambda v
$$

Rearranging:

$$
(A - \lambda I)v = 0
$$

This has a nonzero solution $v$ only if $A - \lambda I$ is singular, i.e.:

$$
\det(A - \lambda I) = 0
$$

This is the **characteristic equation**. Expanding the determinant gives a degree-$n$ polynomial in $\lambda$ (the characteristic polynomial), whose roots are the eigenvalues of $A$. An $n \times n$ matrix has exactly $n$ eigenvalues counted with multiplicity (over $\mathbb{C}$).

## Worked Derivation: A 2×2 Example

Let $A = \begin{pmatrix} 4 & 1 \\ 2 & 3 \end{pmatrix}$.

**Step 1 — characteristic polynomial:**

$$
\det(A - \lambda I) = \det \begin{pmatrix} 4-\lambda & 1 \\ 2 & 3-\lambda \end{pmatrix} = (4-\lambda)(3-\lambda) - 2 = \lambda^2 - 7\lambda + 10
$$

**Step 2 — solve for roots:**

$$
\lambda^2 - 7\lambda + 10 = (\lambda - 5)(\lambda - 2) = 0 \implies \lambda_1 = 5,\ \lambda_2 = 2
$$

**Step 3 — solve $(A - \lambda I)v = 0$ for each eigenvalue.**

For $\lambda_1 = 5$: $\begin{pmatrix} -1 & 1 \\ 2 & -2 \end{pmatrix} v = 0 \implies v_1 = v_2$, so $v^{(1)} = \begin{pmatrix} 1 \\ 1 \end{pmatrix}$.

For $\lambda_2 = 2$: $\begin{pmatrix} 2 & 1 \\ 2 & 1 \end{pmatrix} v = 0 \implies 2v_1 = -v_2$, so $v^{(2)} = \begin{pmatrix} 1 \\ -2 \end{pmatrix}$.

Check: $A v^{(1)} = \begin{pmatrix} 5 \\ 5 \end{pmatrix} = 5 v^{(1)}$ ✓.

## Diagonalization & the Spectral Decomposition

If $A$ has $n$ linearly independent eigenvectors, stack them as columns of $P$ and the eigenvalues on the diagonal of $\Lambda$:

$$
A = P \Lambda P^{-1}
$$

This is why eigen-decomposition matters computationally: powers of $A$ become trivial, since $A^k = P \Lambda^k P^{-1}$ and $\Lambda^k$ is just each diagonal entry raised to the $k$-th power — no repeated matrix multiplication needed.

If $A$ is **real and symmetric** (as covariance matrices always are), the spectral theorem guarantees:

- All eigenvalues are real.
- Eigenvectors corresponding to distinct eigenvalues are orthogonal.
- $A$ can be written as $A = Q \Lambda Q^\top$ with $Q$ orthogonal ($Q^\top Q = I$).

This symmetric case is precisely the one Principal Component Analysis relies on — see [PCA Derivation](/docs/foundations/linear-algebra/pca-derivation) (roadmap) for the full connection.

## Computing Eigenvalues in Practice

Nobody solves the characteristic polynomial by hand for anything bigger than 3×3 — it's numerically unstable for large $n$. Production code uses iterative algorithms instead.

```python
import numpy as np

A = np.array([[4, 1], [2, 3]])
eigenvalues, eigenvectors = np.linalg.eig(A)

print(eigenvalues)   # [5. 2.]
print(eigenvectors)  # columns are the (normalized) eigenvectors
```

Under the hood, `numpy.linalg.eig` uses the **QR algorithm**: repeatedly factor $A = QR$ and reassign $A \leftarrow RQ$. This sequence converges to an upper-triangular (or block-triangular) matrix similar to $A$, whose diagonal reveals the eigenvalues, without ever forming the characteristic polynomial explicitly.

## Complexity

| Method | Time complexity | Notes |
| --- | --- | --- |
| Characteristic polynomial + root finding | Numerically unstable above ~4×4 | Educational use only |
| QR algorithm (dense) | $O(n^3)$ per full decomposition | Standard for dense symmetric/general matrices |
| Power iteration | $O(n^2)$ per iteration | Finds only the dominant eigenvalue; used in PageRank |
| Lanczos / Arnoldi | $O(n^2)$ or less (sparse) | For large sparse matrices, top-$k$ eigenpairs only |

## Common Mistakes

<ProsCons
  pros={[
    "Recognizing that symmetric matrices always have real eigenvalues and orthogonal eigenvectors — this simplifies a huge fraction of ML math.",
    "Using eigenvalues to reason about convergence (e.g., gradient descent step size bounds relate to the largest eigenvalue of the Hessian).",
  ]}
  cons={[
    "Confusing eigenvalues with singular values — they're only the same for symmetric positive semi-definite matrices.",
    "Assuming every matrix is diagonalizable — defective matrices (repeated eigenvalues with too few independent eigenvectors) require the Jordan form instead.",
    "Forgetting that eigenvalues of a non-symmetric real matrix can be complex, even though the matrix itself has only real entries.",
  ]}
/>
