## Intuition

You're standing on a hilly loss surface, fog everywhere, and you can only feel the slope right under your feet. The sanest strategy is: take a small step in whichever direction goes downhill the fastest, then repeat. That's gradient descent — an iterative, purely local algorithm that never needs a map of the whole surface, only what it can measure at the current point.

<Callout type="tip" title="The one idea that matters">
  The negative gradient is, by construction, the direction of steepest local descent. Gradient descent repeatedly takes a small step in that direction: $\theta \leftarrow \theta - \eta \nabla \mathcal{L}(\theta)$. Every variant (SGD, mini-batch, momentum, Adam) is a modification of *how the step is estimated or shaped*, not a departure from this core update.
</Callout>

## Formal Derivation: Why Move Opposite the Gradient?

We want to choose a small step $h$ (with fixed length $\|h\| = \epsilon$) that decreases $\mathcal{L}$ as much as possible. First-order Taylor expansion:

$$
\mathcal{L}(\theta + h) \approx \mathcal{L}(\theta) + \nabla \mathcal{L}(\theta)^\top h
$$

We want to minimize $\nabla \mathcal{L}(\theta)^\top h$ subject to $\|h\| = \epsilon$. By the Cauchy–Schwarz inequality, $\nabla \mathcal{L}(\theta)^\top h \geq -\|\nabla \mathcal{L}(\theta)\| \|h\|$, with equality exactly when $h$ points opposite to $\nabla \mathcal{L}(\theta)$:

$$
h^* = -\epsilon \, \frac{\nabla \mathcal{L}(\theta)}{\|\nabla \mathcal{L}(\theta)\|}
$$

So among *all* directions of fixed length, $-\nabla \mathcal{L}(\theta)$ is provably the one that decreases the first-order approximation of $\mathcal{L}$ the most. Absorbing the step length into a learning rate $\eta > 0$ gives the update rule:

$$
\theta_{t+1} = \theta_t - \eta \, \nabla \mathcal{L}(\theta_t)
$$

**Why this only guarantees *local* improvement, and only for small $\eta$.** The derivation used a first-order (linear) approximation, which is only accurate near $\theta_t$. If $\eta$ is too large, the actual (nonlinear) loss can increase even though the linear model predicted a decrease — this is exactly the failure mode that motivates adaptive and line-search step sizes.

### Convergence rate for convex, $L$-smooth losses

If $\mathcal{L}$ is convex and $L$-smooth (gradient is $L$-Lipschitz: $\|\nabla \mathcal{L}(x) - \nabla \mathcal{L}(y)\| \le L\|x-y\|$), then gradient descent with $\eta = 1/L$ satisfies:

$$
\mathcal{L}(\theta_t) - \mathcal{L}(\theta^*) \le \frac{\|\theta_0 - \theta^*\|^2}{2\eta t} = O\!\left(\frac{1}{t}\right)
$$

If $\mathcal{L}$ is additionally $\mu$-strongly convex, the rate improves to linear (geometric) convergence:

$$
\|\theta_t - \theta^*\|^2 \le \left(1 - \frac{\mu}{L}\right)^t \|\theta_0 - \theta^*\|^2
$$

The ratio $\kappa = L/\mu$ (the **condition number**) controls the rate: poorly-conditioned problems ($\kappa \gg 1$, i.e., a loss surface shaped like a long narrow valley) converge slowly, which is precisely what momentum is designed to fix (see below).

## Batch, Stochastic, and Mini-Batch Gradient Descent

The loss over a dataset of $N$ examples is an average: $\mathcal{L}(\theta) = \frac{1}{N}\sum_{i=1}^N \ell_i(\theta)$, so $\nabla \mathcal{L}(\theta) = \frac{1}{N}\sum_i \nabla \ell_i(\theta)$.

- **Batch (full-batch) gradient descent** computes the exact gradient using all $N$ examples every step. Exact, low-variance direction, but one step costs $O(N)$ gradient evaluations — infeasible for large $N$.
- **Stochastic gradient descent (SGD)** estimates the gradient using a *single* randomly sampled example: $\hat{g}_t = \nabla \ell_{i_t}(\theta_t)$. This is an unbiased estimator ($\mathbb{E}[\hat{g}_t] = \nabla \mathcal{L}(\theta_t)$) but high-variance — the path is noisy, which paradoxically helps escape shallow local minima and saddle points.
- **Mini-batch SGD** averages over a small batch $B$ of size $b$: $\hat{g}_t = \frac{1}{b}\sum_{i \in B} \nabla \ell_i(\theta_t)$. Variance of the estimator scales as $O(1/b)$, and batching lets hardware (GPU/TPU) parallelize the $b$ gradient computations — the practical default for essentially all deep learning training.

## Momentum

Plain (mini-batch) SGD oscillates badly in narrow, curved valleys (high condition number $\kappa$) because the gradient's direction across the narrow axis dominates the useful signal along the shallow axis. **Momentum** accumulates a running (exponentially-weighted) average of past gradients, which reinforces consistent directions and damps oscillating ones:

$$
v_{t+1} = \beta v_t + \nabla \mathcal{L}(\theta_t), \qquad \theta_{t+1} = \theta_t - \eta \, v_{t+1}
$$

with $\beta \in [0, 1)$ (commonly $0.9$). Unrolling the recursion shows $v_t$ is a decaying weighted sum of *all* past gradients:

$$
v_t = \sum_{k=0}^{t} \beta^{t-k} \nabla \mathcal{L}(\theta_k)
$$

Along a direction where the gradient's sign flips every step (the oscillating, high-curvature axis), consecutive terms partially cancel, shrinking the effective step. Along a direction where the gradient is consistently the same sign (the shallow, low-curvature axis we actually want to move along), the terms reinforce, accelerating the effective step. This is why momentum provably improves the convergence rate for ill-conditioned convex quadratics from $O((\kappa-1)/(\kappa+1))^t$ (plain GD) to $O\big((\sqrt{\kappa}-1)/(\sqrt{\kappa}+1)\big)^t$ (Nesterov's accelerated gradient) — replacing $\kappa$ with $\sqrt{\kappa}$ in the convergence exponent.

**Nesterov momentum** is a refinement that evaluates the gradient at the "look-ahead" point $\theta_t - \eta\beta v_t$ rather than at $\theta_t$ itself, effectively correcting the momentum step using information slightly ahead of the current position, which tightens the convergence guarantee further.

This same update rule is exactly what plugs into the per-parameter gradients that [Backpropagation: Full Derivation](/docs/foundations/calculus-optimization/backpropagation-derivation) computes — backprop answers "what is $\nabla \mathcal{L}(\theta)$," and gradient descent (with whichever variant) answers "given that gradient, how do I update $\theta$."

## Code Example

```python
import numpy as np

def sgd_momentum(grad_fn, theta0, lr=0.01, beta=0.9, n_steps=200, batch_sampler=None):
    """
    grad_fn(theta, batch) -> gradient estimate at theta using `batch`.
    batch_sampler() -> yields a mini-batch (or None for full-batch / exact gradient).
    """
    theta = theta0.copy()
    v = np.zeros_like(theta)
    history = [theta.copy()]

    for t in range(n_steps):
        batch = batch_sampler() if batch_sampler else None
        g = grad_fn(theta, batch)
        v = beta * v + g
        theta = theta - lr * v
        history.append(theta.copy())

    return theta, history

# Example: minimize an ill-conditioned quadratic f(x) = 0.5 * x^T A x
A = np.array([[10.0, 0.0], [0.0, 1.0]])  # condition number kappa = 10
grad_fn = lambda theta, batch: A @ theta

theta_final, path = sgd_momentum(grad_fn, np.array([5.0, 5.0]), lr=0.05, beta=0.9, n_steps=100)
print(theta_final)  # converges toward [0, 0]
```

## Complexity

| Variant | Time per step | Gradient variance | Memory (beyond parameters) |
| --- | --- | --- | --- |
| Batch GD | $O(N \cdot d)$ | 0 | $O(d)$ |
| SGD | $O(d)$ | High | $O(d)$ |
| Mini-batch SGD (batch size $b$) | $O(b \cdot d)$, parallelizable across $b$ | $O(1/b)$ | $O(d)$ |
| SGD with momentum | $O(b \cdot d)$ | $O(1/b)$ (smoothed further by averaging) | $O(2d)$ — must store velocity $v$ |

where $d$ is the number of parameters and $N$ the dataset size.
